## 一、设计理念：为什么需要视图绑定机制？

### 1.1 传统做法的表现层问题

```
❌ 传统表现层的问题：

1. 表现与数据强耦合
   ┌─────────────────────────────────────────────────────────┐
   │ public class PlayerEntity : MonoBehaviour {            │
   │     public int ActorId;                                │
   │     public Transform Trans;                           │
   │     // ... 其他表现数据                               │
   │ }                                                      │
   │                                                          │
   │ 问题：表现类直接依赖数据，删除后要手动处理              │
   └─────────────────────────────────────────────────────────┘

2. GameObject 必须同步创建
   ┌─────────────────────────────────────────────────────────┐
   │ var entity = CreateEntity(data);                       │
   │ var go = Instantiate(prefab);  // 必须立即创建          │
   │ Bind(entity, go);                                       │
   │                                                          │
   │ 问题：网络延迟时无法处理                                │
   └─────────────────────────────────────────────────────────┘

3. 重连后刷新困难
   ┌─────────────────────────────────────────────────────────┐
   │ void OnReconnect() {                                   │
   │     // 需要手动遍历所有实体                             │
   │     foreach (var entity in entities) {                 │
   │         CreateView(entity);                            │
   │     }                                                  │
   │ }                                                      │
   │                                                          │
   │ 问题：逻辑复杂，难以维护                                │
   └─────────────────────────────────────────────────────────┘

4. 模型切换困难
   ┌─────────────────────────────────────────────────────────┐
   │ // 切换外观需要重建整个对象                            │
   │ Destroy(go);                                            │
   │ go = Instantiate(newPrefab);                           │
   │                                                          │
   │ 问题：开销大，体验差                                    │
   └─────────────────────────────────────────────────────────┘
```

### 1.2 视图绑定机制的解决方案

```
✅ 视图绑定机制的设计思路：

┌─────────────────────────────────────────────────────────────┐
│                       核心思想                              │
│                                                             │
│   把"数据实体"和"表现对象"完全分离                         │
│   数据实体不持有 GameObject 引用                           │
│   表现对象由独立的绑定系统管理                             │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  传统方式：                                          │  │
│   │  ┌───────────────────────────────────────────┐    │  │
│   │   │  Entity ──────► GameObject              │    │  │
│   │   │   (直接持有)                            │    │  │
│   │   └───────────────────────────────────────────┘    │  │
│   │   // 耦合严重                                      │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  绑定方式：                                          │  │
│   │  ┌───────────────────────────────────────────┐    │  │
│   │   │                                          │    │  │
│   │   │  Entity          Binder          GO     │    │  │
│   │   │    │               │             │      │    │  │
│   │   │    └───────────────┼─────────────┘      │    │  │
│   │   │                    ▼                     │    │  │
│   │   │              延迟创建/切换               │    │  │
│   │   │                                          │    │  │
│   │   └───────────────────────────────────────────┘    │  │
│   │   // 解耦，灵活                                    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   好处：                                                    │
│   - 数据层不关心表现细节                                  │
│   - 表现可以异步创建                                      │
│   - 重连后一键刷新所有视图                                │
│   - 模型切换零开销                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                    视图绑定的三大承诺                          │
│                                                             │
│   1️⃣ 异步                                              │
│      数据来了，表现可以稍后创建                             │
│                                                             │
│   2️⃣ 隔离                                            │
│      数据层和表现层完全解耦                                │
│                                                             │
│   3️⃣ 可刷新                                            │
│      重连后一键重建所有表现                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解视图绑定系统

### 2.1 什么是"视图绑定（View Binding）"？

**通俗解释**：View Binding 是"数据的眼睛"，把数据实体同步到 Unity 的 GameObject。

```
┌─────────────────────────────────────────────────────────────┐
│                    View Binding = 数据的眼睛                 │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  数据层                                              │  │
│   │  ┌───────────────────────────────────────────────┐  │  │
│   │   │  Entity {                                   │  │  │
│   │   │      ActorId = 1001                         │  │  │
│   │   │      Position = (10, 0, 5)                  │  │  │
│   │   │      ModelId = 2001                        │  │  │
│   │   │  }                                         │  │  │
│   │   └───────────────────────────────────────────────┘  │  │
│   │                      │                               │  │
│   │                      ▼                               │  │
│   │  ┌───────────────────────────────────────────────┐  │  │
│   │   │  Binder                                      │  │  │
│   │   │  ├─► 记录映射关系                             │  │  │
│   │   │  ├─► 管理 GameObject 创建/销毁               │  │  │
│   │   │  └─► 处理插值平滑                            │  │  │
│   │   └───────────────────────────────────────────────┘  │  │
│   │                      │                               │  │
│   │                      ▼                               │  │
│   │  ┌───────────────────────────────────────────────┐  │  │
│   │   │  表现层                                      │  │  │
│   │   │  ┌─────────────────────────────────────────┐ │  │  │
│   │   │  │  GameObject "Actor_1001"               │ │  │  │
│   │   │  │  ├─► Transform (位置)                  │ │  │  │
│   │   │  │  ├─► SkinnedMeshRenderer (模型)       │ │  │  │
│   │   │  │  └─► MonoViewHandle (生命周期)         │ │  │  │
│   │   │  └─────────────────────────────────────────┘ │  │  │
│   │   └───────────────────────────────────────────────┘  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **BattleViewBinder** | 绑定器 | 翻译官 |
| **Entity** | 数据实体 | 剧本 |
| **GameObject** | 表现对象 | 演员 |
| **Handle** | 绑定记录 | 演员档案 |
| **Shell** | 外观资源 | 戏服 |
| **MonoViewHandle** | 生命周期监控 | 场务 |

### 4.4 SampleBuffer - 采样缓冲

**通俗解释**：SampleBuffer 是"录像带"，记录一段时间内的位置变化，用于插值平滑。

```csharp
┌─────────────────────────────────────────────────────────────┐
│                    SampleBuffer 结构                          │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   private struct SampleBuffer                        │  │
│   │   {                                                  │  │
│   │       private const int Capacity = 4;               │  │
│   │       private Sample _s0, _s1, _s2, _s3;             │  │
│   │       private int _count;                           │  │
│   │                                                       │  │
│   │       public void Add(double time, in Vector3 pos); │  │
│   │       public bool TryEvaluate(double time,         │  │
│   │           out Vector3 pos);                         │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   │   private struct Sample                              │  │
│   │   {                                                  │  │
│   │       public double Time;                           │  │
│   │       public Vector3 Pos;                           │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   工作原理：                                          │  │
│   │   ─────────────────────────────────────────────     │  │
│   │                                                       │  │
│   │   ┌─────────────────────────────────────────────┐   │  │
│   │   │                                              │   │  │
│   │   │  采样点记录：                                │   │  │
│   │   │                                              │   │  │
│   │   │      ● (t0, p0)                              │   │  │
│   │   │      ● (t1, p1)                              │   │  │
│   │   │      ● (t2, p2)                              │   │  │
│   │   │                                              │   │  │
│   │   │  查询时刻 T：                                │   │  │
│   │   │                                              │   │  │
│   │   │      t0 < T < t1                             │   │  │
│   │   │      t = (T - t0) / (t1 - t0)                │   │  │
│   │   │      p = Lerp(p0, p1, t)  ──► 插值位置      │   │  │
│   │   │                                              │   │  │
│   │   └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   │   ┌─────────────────────────────────────────────┐   │  │
│   │   │                                              │   │  │
│   │   │  环形缓冲（容量=4）：                         │   │  │
│   │   │  - 超过容量时丢弃最旧的                      │   │  │
│   │   │  - 按时间排序插入                           │   │  │
│   │   │  - 查找时二分定位                           │   │  │
│   │   │                                              │   │  │
│   │   └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   插值参数：                                          │  │
│   │   ─────────────────────────────────────────────     │  │
│   │                                                       │  │
│   │   BackTimeTicks = 1                                  │  │
│   │   ├─► 渲染时间 = 逻辑时间 - 1个tick                 │  │
│   │   └─► 显示"过去"的位置，避免抖动                    │  │
│   │                                                       │  │
│   │   MaxLagTicks = 4                                    │  │
│   │   ├─► 最大延迟容忍 = 4个tick                        │  │
│   │   └─► 防止卡顿导致的跳变                            │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
```

## 五、设计模式总结

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Handle Pattern（句柄模式）                            │
│      ───────────────────────────────────────────────        │
│      通过 Handle 间接访问 GameObject，支持延迟创建           │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   Entity ──► Handle ──► GameObject              │   │
│      │                │                                 │   │
│      │                ├──► 可为 null（延迟创建）        │   │
│      │                ├──► 可切换（模型换装）           │   │
│      │                └──► 可销毁（自动清理）           │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Registry Pattern（注册表模式）                        │
│      ─────────────────────────────────────                  │
│      双向索引快速查找                                       │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   _handles: EntityId ──► Handle                 │   │
│      │   _actorIdToEntityId: ActorId ──► EntityId      │   │
│      │                                                   │   │
│      │   好处：                                          │   │
│      │   - O(1) 时间复杂度查找                          │   │
│      │   - 支持多种方式索引                             │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Observer Pattern（观察者模式）                        │
│      ─────────────────────────────────────                  │
│      MonoViewHandle 监听 GO 销毁事件                        │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   GameObject.OnDestroy()                        │   │
│      │         │                                         │   │
│      │         ▼                                         │   │
│      │   MonoViewHandle.OnDestroy()                     │   │
│      │         │                                         │   │
│      │         ▼                                         │   │
│      │   IMonoViewHandleRegistry                         │   │
│      │         │                                         │   │
│      │         ▼                                         │   │
│      │   Binder.OnMonoViewHandleDestroyed()             │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   4️⃣ Ring Buffer Pattern（环形缓冲模式）                    │
│      ─────────────────────────────────────                  │
│      固定容量采样缓冲                                       │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   容量 = 4                                        │   │
```

### 6.2 自定义 ShellLoader 示例

```csharp
// 场景：使用对象池加载表现对象

public sealed class PooledBattleViewShellLoader : IBattleViewShellLoader
{
    private readonly GameObjectPool _pool;
    private readonly Transform _parent;

    public PooledBattleViewShellLoader(GameObjectPool pool, Transform parent)
    {
        _pool = pool;
        _parent = parent;
    }

    public GameObject CreateShellGameObject(int actorId, int modelId)
    {
        var go = _pool.Get(modelId);
        go.transform.SetParent(_parent);
        go.name = $"Actor_{actorId}";
        return go;
    }
}

// 使用
var pool = new GameObjectPool();
var loader = new PooledBattleViewShellLoader(pool, container);
var binder = new BattleViewBinder(vfx, vfxNode, loader);
```

### 6.3 重连刷新示例

```csharp
// 场景：重连后需要刷新所有视图

public void OnReconnected()
{
    // 1. 清理旧绑定
    _binder.Clear();

    // 2. 重新绑定所有实体
    if (_ctx?.EntityWorld != null)
    {
        _ctx.EntityWorld.ForEachAlive(e =>
        {
            _binder.Sync(e, _ctx);
        });
    }

    // 3. 触发插值刷新
    _binder.TickInterpolation(_ctx, 0f);
}
```

### 6.4 监听销毁事件示例

```csharp
// 场景：在销毁时执行自定义逻辑

public void OnEntityDestroyed(EC.EntityId id)
{
    // 1. 清理 EntityLookup
    _ctx?.EntityLookup?.UnbindByEntityId(id);

    // 2. 清理 Binder 中的记录
    _binder?.OnDestroyed(id);

    // 3. 自定义清理逻辑
    if (_someExternalRef.TryGetValue(id, out var ref))
    {
        ref.Cleanup();
        _someExternalRef.Remove(id);
    }
}
```


## 九、插值参数调优指南

### 9.1 参数说明

```
┌─────────────────────────────────────────────────────────────┐
│                    插值参数说明                               │
│                                                             │
│   BackTimeTicks                                             │
│   ──────────────────                                       │
│   类型：float                                               │
│   默认值：1.0f                                              │
│   范围：0.1f ~ 3.0f                                         │
│                                                             │
│   含义：渲染时间相对逻辑时间的回溯量                         │
│   - 值越大：延迟越大，画面越平滑                            │
│   - 值越小：延迟越小，响应越快                             │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │ 逻辑时间 T=100          渲染时间 = 100 - 1 = 99    │  │
│   │ 显示的是 99 时刻的位置                            │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ──────────────────────────────────────────────────────── │
│                                                             │
│   MaxLagTicks                                               │
│   ─────────────────                                         │
│   类型：float                                               │
│   默认值：4.0f                                              │
│   范围：1.0f ~ 10.0f                                        │
│                                                             │
│   含义：最大延迟容忍量                                      │
│   - 值越大：容忍越高，卡顿时不跳变                         │
│   - 值越小：响应越快，但可能抖动                           │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │ 最小渲染时间 = 逻辑时间 - (BackTime + MaxLag)      │  │
│   │ = 100 - (1 + 4) = 95                                 │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
