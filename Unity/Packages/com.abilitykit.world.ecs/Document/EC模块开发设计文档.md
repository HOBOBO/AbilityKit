## 一、设计理念：为什么需要 EC 模块？

### 1.1 传统做法的代码组织问题

```
❌ 传统代码组织的问题：

1. 对象散乱
   ┌─────────────────────────────────────────────────────────┐
   │ var player = new Player();                             │
   │ var enemy = new Enemy();                               │
   │ var projectile = new Projectile();                     │
   │ var effect = new Effect();                             │
   │                                                          │
   │ // 问题：每个对象都是独立的，难以统一管理               │
   │ // - 难以遍历所有"角色"                                │
   │ // - 难以统一销毁                                      │
   │ // - 难以关联关系                                      │
   └─────────────────────────────────────────────────────────┘

2. 组件耦合
   ┌─────────────────────────────────────────────────────────┐
   │ public class Player : MonoBehaviour {                  │
   │     public Transform Trans;                            │
   │     public Health Health;                              │
   │     public Move Move;                                 │
   │     public Attack Attack;                             │
   │     // 问题：所有逻辑堆在一个类里                       │
   │ }                                                      │
   │                                                          │
   │ // 问题：想复用 Move 逻辑？只能复制粘贴                │
   └─────────────────────────────────────────────────────────┘

3. 层级关系混乱
   ┌─────────────────────────────────────────────────────────┐
   │ // 武器挂在角色身上？                                  │
   │ player.transform.Find("Weapon");                       │
   │                                                          │
   │ // 问题：依赖字符串，容易出错                          │
   │ // 问题：难以批量操作                                  │
   └─────────────────────────────────────────────────────────┘

4. 生命周期管理困难
   ┌─────────────────────────────────────────────────────────┐
   │ void OnPlayerDead() {                                 │
   │     Destroy(player);                                   │
   │     Destroy(weapon);                                   │
   │     Destroy(effects);                                  │
   │     // 问题：容易遗漏                                  │
   │ }                                                      │
   └─────────────────────────────────────────────────────────┘
```

│   │   └───────────────────────────────────────────┘    │  │
│   │   // 所有东西绑在一起                                │  │
│   │                                                       │  │

## 二、核心概念：从零理解 EC 系统

### 2.1 什么是"EC"？

**通俗解释**：EC = Entity（实体）+ Component（组件），一种代码组织方式。

```
┌─────────────────────────────────────────────────────────────┐
│                    EC = Entity + Component                   │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  Entity (实体)                                      │  │
│   │  ─────────────────────────────────────────────      │  │
│   │                                                       │  │
│   │  "东西的 ID"，本身不存储数据                          │  │
│   │  类似于身份证号码                                    │  │
│   │                                                       │  │
│   │  ┌─────────────────────────────────────────────┐   │  │
│   │   │  EntityId { Index=0, Version=1 }             │   │  │
│   │   │                                              │   │  │
│   │   │  - Index: 数组下标                            │   │  │
│   │   │  - Version: 版本号，防止复用问题              │   │  │
│   │   └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  Component (组件)                                    │  │
│   │  ───────────────────────────────────────            │  │
│   │                                                       │  │
│   │  "实体的数据"，每个组件负责一类数据                  │  │
│   │  类似于物品清单上的条目                              │  │
│   │                                                       │  │
│   │  ┌─────────────────────────────────────────────┐   │  │
│   │   │  TransformComponent { X, Y, Z }            │   │  │
│   │   │  HealthComponent { HP, MaxHP }              │   │  │
│   │   │  MoveComponent { Speed, Direction }         │   │  │
│   │   └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  EntityWorld (实体世界)                              │  │
│   │  ───────────────────────────────────────            │  │
│   │                                                       │  │
│   │  "仓库管理员"，管理所有实体和组件                    │  │
│   │  类似于物品仓库的管理系统                            │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **Entity** | 实体 | 身份证号码 |
| **EntityId** | 实体标识 | 身份证号（含版本） |
| **Component** | 组件 | 物品清单条目 |
| **EntityWorld** | 实体世界 | 仓库管理员 |
| **ComponentTypeId** | 组件类型ID | 条目分类编号 |

### 3.2 实体创建流程图

```
┌─────────────────────────────────────────────────────────────────────┐
│                      实体创建流程                                      │
│                                                                     │
│   EntityWorld.Create()                                             │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  1. 从空闲池获取索引                                │   │  │
│   │   │                                                     │   │  │
│   │   │  if (_free.Count > 0) {                             │   │  │
│   │   │      index = _free.Pop();  // 复用                  │   │  │
│   │   │  } else {                                           │   │  │
│   │   │      index = AllocateNewIndex();  // 扩容           │   │  │
│   │   │  }                                                   │   │  │
│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │
│   │                               ▼                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  2. 初始化实体数据                                  │   │  │
│   │   │                                                     │   │  │
│   │   │  _alive[index] = true;                             │   │  │
│   │   │  _parentIndex[index] = -1;  // 无父级              │   │  │
│   │   │  _components[index] = null;  // 暂无组件           │   │  │
│   │   │  _children[index]?.Clear();                        │   │  │
│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │

│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │
│   │                               ▼                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  4. 触发事件                                        │   │  │
│   │   │                                                     │   │  │
│   │   │  EntityCreated?.Invoke(e);                         │   │  │
│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │
│   │                               ▼                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  5. 返回实体句柄                                    │   │  │
│   │   │                                                     │   │  │
│   │   │  return e;  // Entity { _world, Id }                │   │  │
│   │   │                                                     │   │  │

### 3.3 实体销毁流程图

```
┌─────────────────────────────────────────────────────────────────────┐
│                      实体销毁流程                                      │
│                                                                     │
│   EntityWorld.Destroy(id)                                           │
│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  1. 检查存活状态                                    │   │  │
│   │   │                                                     │   │  │
│   │   │  EnsureAlive(id);  // 抛异常或直接返回              │   │  │
│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │
│   │                               ▼                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  2. 从父级移除                                      │   │  │
│   │   │                                                     │   │  │
│   │   │  if (_parentIndex[index] != -1) {                   │   │  │
│   │   │      var p = _parentIndex[index];                  │   │  │
│   │   │      _children[p].Remove(index);                   │   │  │
│   │   │      _childIds[p].Remove(...);                     │   │  │
│   │   │  }                                                  │   │  │
│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │
│   │                               ▼                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  3. 递归清理子级                                    │   │  │
│   │   │                                                     │   │  │
│   │   │  if (_children[index] != null) {                   │   │  │
│   │   │      for each child in _children[index]:           │   │  │
│   │   │          _parentIndex[child] = -1;                 │   │  │
│   │   │      _children[index].Clear();                     │   │  │
│   │   │  }                                                  │   │  │
│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │
│   │                               ▼                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  4. 清理组件索引                                    │   │  │
│   │   │                                                     │   │  │
│   │   │  RemoveFromComponentIndex(index, _components);     │   │  │
│   │   │  _components[index] = null;                        │   │  │
│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │
│   │                               ▼                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  5. 标记为死亡                                      │   │  │
│   │   │                                                     │   │  │
│   │   │  _alive[index] = false;                            │   │  │
│   │   │  _versions[index]++;  // 版本+1                   │   │  │
│   │   │  _free.Push(index);  // 放回空闲池                │   │  │
│   │   │                                                     │   │  │
│   │   └─────────────────────────────────────────────────────┘   │  │
│   │                               │                             │  │
│   │                               ▼                             │  │
│   │   ┌─────────────────────────────────────────────────────┐   │  │
│   │   │  6. 触发事件                                        │   │  │

### 4.1 EntityId - 实体标识

**通俗解释**：EntityId 是"带版本号的身份证号"，避免使用已销毁的实体。

```csharp
┌─────────────────────────────────────────────────────────────┐

│   │   }                                                  │  │
│   │                                                       │  │

│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │

│   │   ─────────────────────────────────────────────     │  │
│   │                                                       │  │
│   │   ┌─────────────────────────────────────────────┐   │  │
│   │   │                                              │   │  │
│   │   │  问题：Entity 销毁后，Index 会被复用         │   │  │
│   │   │                                              │   │  │

│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Index Pool Pattern（索引池模式）                      │
│      ─────────────────────────────────────                  │
│      销毁的索引放入池中复用                                  │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   创建：_free.Pop() ──► 获取索引                │   │
│      │   销毁：_free.Push(index) ──► 归还索引          │   │
│      │                                                   │   │
│      │   减少数组扩容，提高内存利用率                   │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Version Pattern（版本模式）                           │
│      ─────────────────────────────────────                  │
│      通过版本号避免悬空引用                                  │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   IsAlive = _alive[index] &&                    │   │
│      │             _versions[index] == id.Version      │   │
│      │                                                   │   │
│      │   销毁时 _versions[index]++                     │   │
│      │   旧引用自动失效                                 │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   4️⃣ Event Bus Pattern（事件总线模式）                     │
│      ─────────────────────────────────────                  │
│      通过事件通知外部系统变化                                │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   EntityWorld                                    │   │
│      │   ├─► EntityCreated                             │   │
│      │   ├─► EntityDestroyed                           │   │
│      │   ├─► ComponentSet                              │   │
│      │   ├─► ComponentRemoved                          │   │
│      │   └─► ParentChanged                             │   │
│      │                                                   │   │
│      │   外部订阅，实现解耦                             │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   5️⃣ Spatial Index Pattern（空间索引模式）                  │
│      ─────────────────────────────────────                  │
│      组件索引加速查询                                       │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   _componentIndex:                              │   │
│      │   typeId ──► HashSet<entityIndex>               │   │
│      │                                                   │   │
│      │   ForEachWith<T>():                             │   │
│      │   1. _componentIndex[typeId] 获取所有实体        │   │
│      │   2. 遍历 HashSet                                │   │
│      │   3. 检查 _alive[index]                         │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

public void DamageAllEnemies(EntityWorld world, float damage)
{
    // 1. 遍历所有有 HealthComponent 的实体
    world.ForEachWith<HealthComponent>((entity, health) =>

    });

### 6.4 事件监听示例

```csharp
// 场景：监听实体创建和销毁

public class EntityTracker
{
    private EntityWorld _world;
    private int _entityCount;

    public EntityTracker(EntityWorld world)
    {
        _world = world;

        // 订阅事件
        _world.EntityCreated += OnEntityCreated;
        _world.EntityDestroyed += OnEntityDestroyed;
        _world.ComponentSet += OnComponentSet;
    }

    private void OnEntityCreated(Entity entity)
    {
        _entityCount++;
        Debug.Log($"Created: {entity.Id}, Total: {_entityCount}");

        // 可以在这里初始化默认组件
        if (!entity.HasComponent<TransformComponent>())
        {
            entity.AddComponent(new TransformComponent());
        }
    }

    private void OnEntityDestroyed(EntityId id)
    {
        _entityCount--;
        Debug.Log($"Destroyed: {id}, Total: {_entityCount}");
    }

    private void OnComponentSet(EntityId id, int typeId, object component)
    {
        Debug.Log($"Component set: {id} -> {component.GetType().Name}");
    }

    public void Dispose()
    {
        _world.EntityCreated -= OnEntityCreated;
        _world.EntityDestroyed -= OnEntityDestroyed;
        _world.ComponentSet -= OnComponentSet;
    }
}
```

        e.AddComponent(new UnityObjectComponent { Behaviour = mb });

    public static Entity WrapAndChild(EntityWorld world, Transform parent,

        var childEntity = parentEntity.AddChild(childId, child.name);

        parentEntity.AddComponent(new UnityObjectComponent { Behaviour = parent });
        childEntity.AddComponent(new UnityObjectComponent { Behaviour = child });

        return parentEntity;
    }
