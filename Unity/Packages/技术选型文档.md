# Ability-Kit 通用战斗框架技术选型文档

> **文档目标**：介绍如何从零开发一个通用战斗框架，涵盖核心设计理念、架构模式、技术实现细节
>
> **适用读者**：希望了解战斗系统设计的技术负责人、框架开发者

## 一、设计理念：从业务问题出发

### 1.1 通用战斗框架的核心挑战

在游戏开发中，战斗系统是最复杂也是最具复用价值的模块之一。一个设计良好的通用战斗框架需要解决以下核心问题：

**1. 逻辑与表现的高效分离**

- 战斗逻辑需要在服务器、客户端、编辑器环境下运行
- 表现层（动画、特效、UI）需要与逻辑解耦，避免相互污染
- 逻辑层必须保持纯 C#，不能依赖 Unity 引擎 API

**2. 高度可组合的技能系统**

- 技能是战斗系统的核心原子单元
- 技能需要支持多种执行模式：瞬发、引导、充能、吟唱
- 技能效果需要可组合、可配置、可追溯

**3. 数据驱动的开发模式**

- 策划需要能够通过 Excel/JSON 配置技能和行为
- 配置变更应该快速生效，无需代码改动
- 需要支持复杂的效果链和条件判断

**4. 网络同步与一致性**

- MOBA/格斗类游戏对帧同步有严格需求
- 需要支持预测、回滚、断线重连
- 逻辑层必须完全确定性

**5. 可测试性与可调试性**

- 业务逻辑需要能够在单元测试中验证
- 需要提供技能执行过程的完整日志和可视化
- 框架需要提供诊断工具排查运行时问题

### 1.2 Ability-Kit 的设计目标

Ability-Kit 定位为**通用战斗框架**，它不是为某一款游戏量身定制，而是抽象出战斗系统的共性需求，提供可复用的基础设施：

| 目标 | 说明 |
|------|------|
| **跨运行时** | 支持 Unity 运行时、编辑器仿真、服务器端 |
| **高度可扩展** | 通过模块化设计，允许按需裁剪和扩展 |
| **数据驱动** | 技能、效果、触发器均可通过配置定义 |
| **确定性强** | 保证逻辑层的完全确定性，支持帧同步 |
| **工具链完善** | 提供编辑器、调试器、测试工具 |

## 三、逻辑与表现分离：双 Runtime 架构

### 3.1 问题背景

传统战斗框架中，逻辑代码和表现代码往往混杂在一起，导致：

- **测试困难**：无法在服务器环境运行测试
- **复用性差**：逻辑和表现强耦合
- **维护成本高**：修改表现可能影响逻辑

### 3.2 解决方案：严格的 Runtime 分离

Ability-Kit 强制将代码分为两个独立的 Runtime：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    com.abilitykit.demo.moba.runtime                 │
├─────────────────────────────────────────────────────────────────────┤
│  目标：纯 C#，无 Unity 依赖                                           │
│                                                                     │
│  包含：                                                             │
│  • 技能逻辑 (SkillLogic)                                            │
│  • 伤害计算 (DamageService)                                         │
│  • Buff 状态管理 (BuffSystem)                                        │
│  • AI 决策 (AIController)                                           │
│  • 实体数据 (EntityData)                                             │
│  • 世界状态 (MobaWorld)                                              │
│                                                                     │
│  特点：                                                              │
│  ✓ 可在 .NET 单元测试中运行                                         │
│  ✓ 可在服务器端运行                                                 │
│  ✓ 帧同步确定性保证                                                 │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│                 com.abilitykit.demo.moba.view.runtime                │

│  包含：                                                              │
│  • Unity 单位表现 (BattleUnitView)                                  │
│  • 动画播放 (AnimationController)                                    │
│  • 特效/音效 (VFXManager, AudioManager)                              │
```

### 3.4 关键设计原则

| 原则 | 说明 | 违反后果 |
|------|------|----------|
| **逻辑层禁止引用 Unity 类型** | 不能使用 `GameObject`、`Transform`、`MonoBehaviour` | 无法跨平台运行 |
| **表现层禁止执行业务逻辑** | 只负责渲染和输入采集 | 破坏帧同步确定性 |
| **通过接口通信** | 避免直接类型引用 | 增加耦合 |
| **事件驱动而非直接调用** | 逻辑层不知道表现层存在 | 紧耦合 |
| **数据对象使用纯 C# 类型** | `Vec3` 而非 `UnityEngine.Vector3` | 破坏确定性 |

## 四、管线系统：技能执行的可扩展编排

### 4.1 问题背景

游戏中技能的执行模式多种多样：

- **瞬发技能**：无前摇，释放即生效
- **引导技能**：持续施法，期间可被打断
- **充能技能**：需要积累能量后释放
- **蓄力技能**：按住蓄力，松手释放
- **连招技能**：多个阶段顺序执行

传统实现方式（状态机或硬编码）难以优雅地处理这种复杂性。

### 4.2 解决方案：Pipeline + Phase 模式

Ability-Kit 引入了 **管线系统**，灵感来自数据处理管线和 Unity 的 Timeline：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         AbilityPipeline                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Phase1 (PreCast)      Phase2 (Cast)        Phase3 (AfterCast)      │
│  ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐   │
│  │ 前摇/引导阶段   │──▶│ 施法阶段         │──▶│ 后摇阶段        │   │
│  │                 │   │                 │   │                 │   │
│  │ • 显示引导特效  │   │ • 触发效果      │   │ • 冷却计时      │   │
│  │ • 检测目标      │   │ • 投射物生成    │   │ • 资源恢复      │   │
│  │ • 可被打断      │   │ • 伤害结算      │   │                 │   │
│  └─────────────────┘   └─────────────────┘   └─────────────────┘   │
│                                                                     │
│  生命周期：                                                         │
│  Start() → Tick(deltaTime) → Complete() / Interrupted() / Failed() │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.3 阶段类型体系

```csharp
// 阶段基类
public abstract class AbilityPipelinePhaseBase<TCtx>
{
    public abstract bool ShouldExecute(TCtx context);
    public abstract void Execute(TCtx context);
    public abstract void OnUpdate(float deltaTime, TCtx context);
    public abstract bool IsComplete(TCtx context);
    public abstract PhaseResult OnComplete(TCtx context);
}

// 瞬时阶段：在一帧内完成
public abstract class AbilityInstantPhaseBase<TCtx> : AbilityPipelinePhaseBase<TCtx>
{
    public sealed override void Execute(TCtx context) => OnInstantExecute(context);
    public sealed override void OnUpdate(float deltaTime, TCtx context) { }
    public sealed override bool IsComplete(TCtx context) => true;
    
    protected abstract void OnInstantExecute(TCtx context);
}

// 持续阶段：基于时间执行
public abstract class AbilityDurationalPhaseBase<TCtx> : AbilityPipelinePhaseBase<TCtx>
{
    protected float Duration;           // 持续时间
    protected float ElapsedTime;       // 已消耗时间
    
    public sealed override void Execute(TCtx context)
    {
        ElapsedTime = 0;
        OnEnter(context);
    }
    
    public sealed override void OnUpdate(float deltaTime, TCtx context)
    {
        ElapsedTime += deltaTime;
        OnTick(deltaTime, context);
    }
    
    public sealed override bool IsComplete(TCtx context) => ElapsedTime >= Duration;
    
    protected virtual void OnEnter(TCtx context) { }
    protected virtual void OnTick(float deltaTime, TCtx context) { }
    protected virtual void OnExit(TCtx context) { }
}

// 可中断阶段：支持外部中断
public abstract class AbilityInterruptiblePhaseBase<TCtx> : AbilityPipelinePhaseBase<TCtx>
{
    public sealed override PhaseResult OnComplete(TCtx context)
    {
        OnInterrupt(context);
        return PhaseResult.Interrupted;
    }
    
    protected abstract void OnInterrupt(TCtx context);
}
```

### 4.4 复合阶段：支持复杂流程

```csharp
// 顺序执行：子阶段按顺序执行
public class AbilitySequencePhase<TCtx> : AbilityCompositePhase<TCtx>
{
    private int _currentIndex;
    
    public override void Execute(TCtx context)
    {
        _currentIndex = 0;
        EnterChild(_currentIndex, context);
    }
    
    public override void OnChildComplete(TCtx context)
    {
        _currentIndex++;
        if (_currentIndex < Children.Count)
        {
            EnterChild(_currentIndex, context);
        }
        else
        {
            Complete(context);
        }
    }
}

// 并行执行：子阶段同时运行
public class AbilityParallelPhase<TCtx> : AbilityCompositePhase<TCtx>
{
    public override void Execute(TCtx context)
    {
        foreach (var child in Children)
        {
            EnterChild(child, context);
        }
    }
    
    public override void OnChildComplete(TCtx context)
    {
        // 当所有子阶段完成时，整个阶段完成
        if (Children.All(c => c.IsComplete(context)))
        {
            Complete(context);
        }
    }
}

// 条件分支
public class AbilityConditionalPhase<TCtx> : AbilityPipelinePhaseBase<TCtx>
{
    public Func<TCtx, int> Selector;  // 返回分支索引
    public List<AbilityPipelinePhaseBase<TCtx>> Branches;
    
    public override void Execute(TCtx context)
    {
        var branchIndex = Selector(context);
        if (branchIndex >= 0 && branchIndex < Branches.Count)
        {
            EnterChild(Branches[branchIndex], context);
        }
    }
}
```

### 4.5 技能管线执行器

```csharp
public sealed class SkillPipelineRunner
{
    private readonly List<SkillPipelineEntry> _entries = new List<SkillPipelineEntry>();
    
    public bool TryQueue(SkillCommand command)
    {
        // 查找匹配的技能配置
        var skillDef = _skillLibrary.GetSkill(command.SkillId);
        if (!CanCast(skillDef, command.Caster)) return false;
        
        // 创建技能上下文
        var context = new SkillCastContext
        {
            SkillId = command.SkillId,
            CasterActorId = command.CasterId,
            AimPos = command.AimPosition,
            WorldServices = _services,
        };
        
        // 启动管线
        var pipeline = _pipelineFactory.Create(skillDef, context);
        _entries.Add(new SkillPipelineEntry { Pipeline = pipeline, Context = context });
        
        return true;
    }
    
    public void Tick(float deltaTime)
    {
        for (int i = _entries.Count - 1; i >= 0; i--)
        {
            var entry = _entries[i];
            entry.Pipeline.Step(deltaTime);
            
            if (entry.Pipeline.IsComplete)
            {
                _entries.RemoveAt(i);
            }
        }
    }
}
```

## 五、ECS 架构：数据与逻辑的分离

### 5.1 问题背景

在战斗系统中，我们经常需要：

- 查询所有「血量低于 30% 且未被沉默」的单位
- 查询「施法中的技能」对应的释放者
- 查询「某个阵营的所有单位」

传统面向对象方式会导致：

- 继承层次过深（`Entity` → `Unit` → `Hero` → `MobaHero`）
- 查询逻辑分散在各个类中
- 难以优化性能

### 5.2 解决方案：轻量级 ECS

Ability-Kit 提供了轻量级 ECS 框架：

```csharp
// 实体定义
public readonly struct Entity : IEquatable<Entity>
{
    private readonly EntityWorld _world;
    public readonly EntityId Id;
    
    public Entity(EntityWorld world, EntityId id)
    {
        _world = world;
        Id = id;
    }
    
    public bool IsAlive => _world.IsAlive(Id);
    
    public void AddComponent<T>(T component) where T : class
        => _world.AddComponent(Id, component);
    
    public T GetComponent<T>() where T : class
        => _world.GetComponent<T>(Id);
    
    public bool TryGetComponent<T>(out T component) where T : class
        => _world.TryGetComponent(Id, out component);
    
    public bool HasComponent<T>() where T : class
        => _world.HasComponent<T>(Id);
    
    public void RemoveComponent<T>() where T : class
        => _world.RemoveComponent<T>(Id);
    
    public void Destroy() => _world.DestroyEntity(Id);
}

// 世界管理器
public sealed class EntityWorld
{
    private int[] _versions;
    private bool[] _alive;
    private object[][] _components;
    private int _entityCount;
    
    public Entity Create()
    {
        var id = new EntityId(_entityCount++, AcquireVersion());
        _alive[id.Value] = true;
        return new Entity(this, id);
    }
    
    // 高效的组件查询
    public void ForEachWith<T>(Action<Entity, T> visitor)
    {
        var pool = GetPool<T>();
        foreach (var id in pool)
        {
            if (IsAlive(id))
            {
                visitor(new Entity(this, id), pool[id]);
            }
        }
    }
    
    // 组合查询
    public void ForEachWith<T1, T2>(Action<Entity, T1, T2> visitor)
    {
        var pool1 = GetPool<T1>();
        var pool2 = GetPool<T2>();
        
        foreach (var id in pool1)
        {
            if (IsAlive(id) && pool2.Has(id))
            {
                visitor(new Entity(this, id), pool1[id], pool2[id]);
            }
        }
    }
}
```

### 5.4 系统设计

```csharp
// 系统基类
public abstract class EntitySystem
{
    public abstract void Execute(EntityWorld world);
}

// 伤害结算系统
public class DamageSystem : EntitySystem
{
    public override void Execute(EntityWorld world)
    {
        world.ForEachWith<CombatComponent, DamageRequestComponent>(
            (entity, combat, request) =>
            {
                var damage = CalculateDamage(request);
                combat.CurrentHp -= damage;
                
                if (combat.CurrentHp <= 0)
                {
                    entity.AddComponent(new DeathComponent());
                }
                
                entity.RemoveComponent<DamageRequestComponent>();
            });
    }

    {
        world.ForEachWith<PositionComponent, MoveTargetComponent>(
            (entity, pos, target) =>
            {
                var direction = (target.Position - pos.Value).Normalized;
                pos.Value += direction * target.Speed * Time.deltaTime;
            });
    }
}
```

```csharp
// 自动选择参数最多的构造函数
public static object Create(Type implType, IWorldResolver resolver)
{
    var plan = GetOrBuildPlan(implType);
    
    foreach (var ctor in plan.Constructors.OrderByDescending(c => c.ParameterCount))
    {
        var args = new object[ctor.ParameterCount];
        var canResolve = true;
        
        foreach (var param in ctor.Parameters)
        {
            if (resolver.TryResolve(param.Type, out var arg))
            {
                args[param.Index] = arg;
            }
            else
            {
                canResolve = false;
                break;
            }
        }
        
        if (canResolve)
        {
            return ctor.Invoke(args);
        }
    }
    
    throw new InvalidOperationException($"No suitable constructor for {implType}");
}
```

## 七、触发器系统：事件驱动解耦

### 7.1 问题背景

技能执行过程中，需要响应各种事件：

- 伤害结算完成后 → 触发吸血效果
- 技能释放完成 → 触发冷却
- 单位死亡 → 触发复活效果
- Buff 叠加到指定层数 → 触发特殊效果

传统方式（回调/委托链）会导致：

- 代码耦合严重
- 难以追踪触发链路
- 条件逻辑分散

### 7.2 解决方案：声明式触发器

Ability-Kit 实现了可配置的事件触发器系统：

```csharp
// 触发器定义
public interface ITrigger<TArgs, TCtx>
{
    bool CanExecute(TArgs args, TCtx context);
    void Execute(TArgs args, TCtx context);
}

// 触发器计划（编译后的配置）
public readonly struct TriggerPlan<TArgs>
{
    public readonly int Phase;           // 执行阶段
    public readonly int Priority;       // 优先级
    public readonly PredicatePlan Predicate;  // 条件计划
    public readonly ActionPlan[] Actions;     // 执行动作
}

// 触发器运行器
public sealed class TriggerRunner<TCtx>
{
    private readonly Dictionary<Type, List<TriggerRegistration>> _triggers = new();
    
    // 注册触发器
    public IDisposable Register<TArgs>(
        EventKey<TArgs> key,
        ITrigger<TArgs, TCtx> trigger,
        int priority = 0)
    {
        var registration = new TriggerRegistration(trigger, priority);
        _triggers[typeof(TArgs)].Add(registration);
        return new Disposable(() => _triggers[typeof(TArgs)].Remove(registration));
    }
    
    // 分发事件并执行触发器
    public void Dispatch<TArgs>(TArgs args, TCtx context)
    {
        var handlers = _triggers[typeof(TArgs)];
        foreach (var handler in handlers.OrderByDescending(h => h.Priority))
        {
            if (handler.Trigger.CanExecute(args, context))
            {
                handler.Trigger.Execute(args, context);
            }
        }
    }
}
```

```csharp
public sealed class RollbackCoordinator
{
    private readonly RollbackRegistry _registry;
    private readonly RollbackSnapshotBuffer _buffer;
    
    // 快照存储
    public bool CaptureAndStore(FrameIndex frame)
    {
        var snapshot = new WorldRollbackSnapshot
        {
            Frame = frame,
            Version = ++_snapshotVersion,
            Entries = _registry.CaptureAll()
        };
        return _buffer.Store(frame, snapshot);
    }
    
    // 回滚到指定帧
    public bool TryRestore(FrameIndex frame)
    {
        if (!_buffer.TryGet(frame, out var snapshot))
            return false;
        
        _registry.Restore(snapshot.Entries);
        return true;
    }
}

// 快照缓冲区（环形缓冲区）
public sealed class RollbackSnapshotBuffer
{
    private readonly RollbackSnapshot[] _buffer;
    private readonly int _capacity;
    private int _head;
    
    public bool Store(FrameIndex frame, RollbackSnapshot snapshot)
    {
        var index = (int)(frame.Value % _capacity);
        _buffer[index] = snapshot;
        _head = index;
        return true;
    }
    
    public bool TryGet(FrameIndex frame, out RollbackSnapshot snapshot)
    {
        var index = (int)(frame.Value % _capacity);
        snapshot = _buffer[index];
        return snapshot != null && snapshot.Frame == frame;
    }
}
```

```csharp
// 客户端预测执行器
public sealed class ClientPrediction
{
    private readonly RollbackCoordinator _rollback;
    private readonly InputBuffer _inputBuffer;
    private FrameIndex _localFrame;
    
    public void ExecutePredictedCommand(FrameCommand command)
    {
        // 立即在本地执行预测
        _world.Execute(command);
        _localFrame = command.Frame;
        
        // 保存输入记录用于回滚
        _inputBuffer.Record(command);
    }
    
    // 服务器确认后回滚
    public void OnServerConfirm(FrameIndex confirmedFrame, WorldRollbackSnapshot snapshot)
    {
        if (confirmedFrame < _localFrame)
        {
            // 需要回滚
            _rollback.TryRestore(confirmedFrame);
            
            // 重放丢失的帧
            ReplayMissedFrames(confirmedFrame + 1, _localFrame);
        }
    }
    
    private void ReplayMissedFrames(FrameIndex from, FrameIndex to)
    {
        for (var frame = from; frame <= to; frame++)
        {
            var command = _inputBuffer.GetCommand(frame);
            if (command != null)
            {
                _world.Execute(command);
            }
        }
    }
}
```

## 十一、总结

Ability-Kit 通过以下核心技术实现了通用战斗框架：

| 技术点 | 实现方式 | 解决的问题 |
|--------|----------|------------|
| **逻辑与表现分离** | 双 Runtime 架构 | 测试困难、耦合严重 |
| **管线系统** | Phase 组合模式 | 技能执行复杂性 |
| **ECS 架构** | 轻量级 EntityWorld | 高效查询、组件复用 |
| **依赖注入** | WorldScope 服务定位器 | 依赖管理、测试困难 |
| **事件触发器** | TriggerRunner | 紧耦合、难以追踪 |
| **数据驱动** | Excel → ScriptableObject | 策划效率低 |
| **帧同步** | Rollback + Prediction | 网络延迟、断线重连 |

这些设计并非独创，而是融合了业界最佳实践：

- 管线系统借鉴了数据处理引擎的设计
- ECS 参考了 Entitas 和 Unity ECS 的理念
- 依赖注入借鉴了 ASP.NET Core 的设计
- 帧同步参考了 Roll4 和 FoxFS 的实现
