## 一、设计理念：为什么要做投射物系统模块？

### 1.1 传统做法的问题

```
❌ 传统做法的投射物问题：

1. 每帧创建大量临时对象
   ┌─────────────────────────────────────────────────────────┐
   │ // 弓箭、子弹每次发射都 new                             │
   │ void Shoot() {                                        │
   │     var bullet = new GameObject("Bullet");            │
   │     bullet.AddComponent<Rigidbody>();                  │
   │     bullet.AddComponent<Collider>();                  │
   │     // ... 配置代码                                    │
   │     // 发射后等待销毁，或者忘记销毁                   │
   │ }                                                     │
   │                                                          │
   │ // 每秒发射10个子弹 → 每秒10个GC分配                   │
   │ // 100个玩家同时发射 → 大量GC                          │
   └─────────────────────────────────────────────────────────┘

2. 碰撞检测逻辑散落各处
   ┌─────────────────────────────────────────────────────────┐
   │ // 每个技能都写一套碰撞逻辑                            │
   │ void Update() {                                        │
   │     if (hitSomething) {                               │
   │         ApplyDamage();                                  │
   │         CreateEffect();                                 │
   │         PlaySound();                                   │
   │         // 10个技能就重复10次                         │
   │     }                                                  │
   │ }                                                     │
   │                                                          │
   │ // 改一处要改N处                                       │
   └─────────────────────────────────────────────────────────┘

3. 网络同步困难
   ┌─────────────────────────────────────────────────────────┐
   │ // 客户端和服务端结果不一致                            │
   │ // 浮点数精度差异                                      │
   │ // 碰撞检测时间差异                                    │
   │ // 回放验证困难                                        │
   └─────────────────────────────────────────────────────────┘

4. 不同技能需要不同行为
   ┌─────────────────────────────────────────────────────────┐
   │ // 箭矢：击中后消失                                   │
   │ // 穿透箭：穿透多个目标                               │
   │ // 回旋镖：击中后返回                                  │
   │ // 激光：持续伤害                                     │
   │                                                          │
   │ // 每种都要单独写代码                                  │
   └─────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解投射物系统

### 2.1 什么是"投射物（Projectile）"？

**通俗解释**：Projectile 是"飞行的子弹"，包括移动、碰撞检测和命中事件。

```
┌─────────────────────────────────────────────────────────────┐
│                    Projectile = 飞行的子弹                   │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   属性：                                              │  │
│   │   ├─► Position      当前位置                        │  │
│   │   ├─► Direction     飞行方向                        │  │
│   │   ├─► Speed         飞行速度                        │  │
│   │   ├─► OwnerId       所属者                         │  │
│   │   ├─► LayerMask    碰撞层                          │  │
│   │   └─► HitsRemaining 剩余可命中次数                  │  │
│   │                                                       │  │
│   │   行为：                                              │  │
│   │   ├─► 每帧移动 Speed * dt 的距离                   │  │
│   │   ├─► 碰撞检测                                      │  │
│   │   └─► 命中时触发事件                                │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   生命周期：                                          │  │
│   │                                                       │  │
│   │   Spawn ──► Tick ──► Tick ──► ... ──► Exit         │  │
│   │     │       │       │                    │         │  │
│   │     │       │       │                    ▼         │  │
│   │     │       │       │              HitEvent        │  │
│   │     │       │       │              ExitEvent      │  │
│   │     │       │       │                               │  │
│   │     │       │       ▼                               │  │
│   │     │       │    TickEvent                          │  │
│   │     │       ▼                                       │  │
│   │     │    HitEvent (可能多次)                        │  │
│   │     ▼                                              │  │
│   │   SpawnEvent                                        │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **ProjectileWorld** | 投射物世界 | 子弹工厂 |
| **Projectile** | 具体的投射物实例 | 一颗子弹 |
| **ProjectileService** | 投射物服务 | 子弹管理系统 |
| **IProjectileHitPolicy** | 命中策略 | 子弹效果（消失/穿透） |
| **IProjectileHitFilter** | 命中过滤器 | 友军免伤 |
| **IProjectileSpawnPattern** | 发射模式 | 霰弹/连射 |
| **AreaWorld** | 范围效果世界 | 地雷区 |
| **ProjectileSchedule** | 定时发射计划 | 定时炸弹 |

## 四、核心组件详解

### 4.1 ProjectileService - 投射物服务

**通俗解释**：Service 是"子弹管理系统"，提供统一的发射、查询、事件接口。

```
┌─────────────────────────────────────────────────────────────┐
│                    ProjectileService 结构                       │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   public interface IProjectileService : IService      │  │
│   │   {                                                  │  │
│   │       ProjectileId Spawn(in ProjectileSpawnParams p);  │  │
│   │       bool Despawn(ProjectileId id);                 │  │
│   │       void Tick(int frame, float dt);                 │  │
```

## 五、设计模式总结

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Object Pool Pattern（对象池模式）                     │
│      ─────────────────────────────────────                  │
│      投射物和范围效果预分配，复用对象                       │
│      避免 GC                                                │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   Pool<Projectile>                               │   │
│      │   ├─► Acquire()  获取对象                       │   │
│      │   └─► Release()  归还对象                       │   │
│      │                                                   │   │
│      │   // 无需每帧 new/destroy                       │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Strategy Pattern（策略模式）                          │
│      ─────────────────────────────────────                  │
│      IProjectileHitPolicy 定义命中行为                     │
│      IProjectileHitFilter 定义命中过滤                     │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   IProjectileHitPolicy:                          │   │
│      │   ├─► ExitOnHitPolicy    消失                  │   │
│      │   ├─► PierceHitPolicy    穿透                  │   │
│      │   └─► ChainPolicy        链式                  │   │
│      │                                                   │   │
│      │   IProjectileHitFilter:                         │   │
│      │   └─► 自定义过滤逻辑                            │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Builder Pattern（建造者模式）                         │
│      ─────────────────────────────────────                  │
│      IProjectileSpawnPattern 构建发射参数                  │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   SingleShotPattern / BurstPattern / FanPattern │   │
│      │   └─► Build(baseSpawn, results)                │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   4️⃣ Event-Driven Pattern（事件驱动模式）                 │
│      ────────────────────────────────────────────────────  │
│      所有状态变化产生事件，消费者按需处理                   │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   DrainXXXEvents → 消费                        │   │
│      │   PeekXXXEvents  → 观察                        │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   5️⃣ Double Buffer Pattern（双缓冲模式）                   │
│      ─────────────────────────────────────                  │
│      Area 的进入/离开检测使用双缓冲                         │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   prevBuffer  ←────  swap  ────→  currBuffer    │   │
│      │                                                   │   │
│      │   // 比较两个缓冲区检测进入/离开                  │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 各模式解决的问题

| 模式 | 解决的问题 |
|------|------------|
| Object Pool | GC 压力、对象创建开销 |
| Strategy | 灵活替换命中行为 |
| Builder | 灵活构建发射参数 |
| Event-Driven | 解耦模拟和效果 |
| Double Buffer | 帧间状态对比 |

```csharp
public void ProcessAreaEvents()
{
    _projectileService.DrainAreaEnterEvents(_areaEnterEvents);
    foreach (var enter in _areaEnterEvents)
    {
        // 进入时应用初始伤害
        ApplyDamage(enter.OwnerId, enter.ColliderId, _initialDamage);
    }

    _projectileService.DrainAreaStayEvents(_areaStayEvents);
    foreach (var stay in _areaStayEvents)
    {
        // 持续伤害
        ApplyDamage(stay.OwnerId, stay.ColliderId, _tickDamage);
    }

    _projectileService.DrainAreaExitEvents(_areaExitEvents);
    foreach (var exit in _areaExitEvents)
    {
        // 离开时移除燃烧效果
        RemoveBurningEffect(exit.ColliderId);
    }
}
```
