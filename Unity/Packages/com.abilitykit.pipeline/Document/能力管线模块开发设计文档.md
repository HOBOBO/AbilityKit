> **文档目标**：让你理解 Pipeline 模块"是什么"、"解决了什么问题"、"为什么这样设计"、"怎么使用"

## 一、设计理念：为什么要做 Pipeline 模块？

### 1.1 传统能力系统的痛点

```
❌ 传统做法的问题：

1. 技能逻辑散落各处
   ┌─────────────────────────────────────────────────────────┐
   │ // 技能A的逻辑分布在多个地方：                        │
   │ void OnClick() { _isCasting = true; }                 │
   │ void Update() {                                       │
   │     if (_isCasting) { _castTime += dt; ... }        │
   │ }                                                     │
   │ void OnAnimationEvent() { DealDamage(); }             │
   │ void OnInterrupt() { _isCasting = false; ... }       │
   │ // 改一个技能要改4个地方，漏一个就出Bug               │
   └─────────────────────────────────────────────────────────┘

2. 阶段管理混乱
   ┌─────────────────────────────────────────────────────────┐
   │ // 技能阶段：引导 → 施放 → 冷却                       │
   │ // 但这些阶段互相干扰：                                │
   │ // - 引导时被打断？                                   │
   │ // - 施放时移动了？                                   │
   │ // - 冷却时还能施放？                                 │
   │ // 每个地方都要写 if (_isCasting) return;             │
   └─────────────────────────────────────────────────────────┘

3. 难以复用和组合
   ┌─────────────────────────────────────────────────────────┐
   │ // "冲锋"和"突刺"都有"位移"                         │
   │ // 但代码不能复用                                      │
   │ // 新技能只能复制粘贴，改一处忘一处                   │
   └─────────────────────────────────────────────────────────┘

4. 难以调试和追踪
   ┌─────────────────────────────────────────────────────────┐
   │ // "技能为什么没施放？"                               │
   │ // 打断点？不知道在哪设                               │
   │ // 看日志？状态太多跟踪不过来                        │
   └─────────────────────────────────────────────────────────┘
```

### 1.2 Pipeline 模块的解决方案

```
✅ Pipeline 的设计思路：

┌─────────────────────────────────────────────────────────────┐
│                       核心思想                              │
│                                                             │
│   把"技能"抽象成"管线"，每个阶段是一个节点                │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   技能管线示例：                                      │  │
│   │                                                       │  │
│   │   Sequence (顺序执行)                                 │  │
│   │   │                                                   │  │
│   │   ├─► [0] 引导阶段 (0.5秒)                           │  │
│   │   │                                                   │  │
│   │   ├─► [1] 检测条件 (敌人是否存在?)                   │  │
│   │   │     ├─► 存在 → 继续                             │  │
│   │   │     └─► 不存在 → 等待                            │  │
│   │   │                                                   │  │
│   │   ├─► [2] 并行阶段                                   │  │
│   │   │     ├─► 播放特效                                 │  │
│   │   │     ├─► 播放音效                                 │  │
│   │   │     └─► 计算伤害                                 │  │
│   │   │                                                   │  │
│   │   └─► [3] 冷却阶段                                   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                    Pipeline 模块的三大承诺                    │
│                                                             │
│   1️⃣ 阶段清晰                                            │
│      每个阶段做什么、什么时候开始结束，清清楚楚            │
│      不会互相干扰                                         │
│                                                             │
│   2️⃣ 可组合复用                                          │
│      阶段是独立的，可以自由组合                            │
│      搭积木一样构建复杂技能                               │
│                                                             │
│   3️⃣ 可调试追踪                                          │
│      每个阶段的执行都有事件通知                          │
│      完整的状态追踪和错误处理                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解 Pipeline

### 2.1 什么是"管线（Pipeline）"？

**通俗解释**：管线就像一条"生产线"，技能的不同阶段是生产线上的不同工位。

```
┌─────────────────────────────────────────────────────────────┐
│                    管线 = 生产线                            │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   输入原料 ──► [工位1] ──► [工位2] ──► [工位3] ──► 输出  │  │
│   │             引导      检测     执行      冷却           │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   工位1 完成 → 工位2 开始                                  │
│   工位2 完成 → 工位3 开始                                  │
│   工位3 完成 → 生产线结束                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 什么是"阶段（Phase）"？

**通俗解释**：阶段是管线中的一个"工位"，负责完成特定的任务。

```
┌─────────────────────────────────────────────────────────────┐
│                    阶段的三种类型                            │
│                                                             │
│   1️⃣ 瞬时阶段 (Instant Phase)                              │
│   ──────────────────────────────────────────────────────  │
│   ┌───────────────────────────────────────────────────┐   │
│   │  Execute() ──► 立即完成 ──► 下一个阶段           │   │
│   └───────────────────────────────────────────────────┘   │
│   示例：检测条件、播放音效                                  │
│                                                             │
│   2️⃣ 持续阶段 (Durational Phase)                          │
│   ──────────────────────────────────────────────────────  │
│   ┌───────────────────────────────────────────────────┐   │
│   │  Execute() ──► Update() ──► Update() ──► ... ──► 完成  │   │
│   │             │                                    │   │
│   │             ▼                                    │   │
│   │         每帧调用，直到满足条件                    │   │
│   └───────────────────────────────────────────────────┘   │
│   示例：引导吟唱、持续施法                                  │
│                                                             │
│   3️⃣ 可中断阶段 (Interruptible Phase)                      │
│   ──────────────────────────────────────────────────────  │
│   ┌───────────────────────────────────────────────────┐   │
│   │  Execute() ──► Update() ──► Interrupt() ──► 结束  │   │
│   │                           │                       │   │
│   │                           └─► 中断清理            │   │
│   └───────────────────────────────────────────────────┘   │
│   示例：引导被打断                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **Pipeline** | 一条完整的生产线 | 工厂流水线 |
| **Phase** | 生产线上的一个工位 | 装配工 |
| **Run** | 一次生产执行 | 一件产品的生产过程 |
| **Context** | 生产所需的上下文数据 | 原料和工具箱 |
| **Sequence** | 按顺序执行多个阶段 | 流水线 |
| **Parallel** | 同时执行多个阶段 | 多工位同时加工 |
| **Conditional** | 根据条件选择阶段 | 分支传送带 |

### 3.2 管线执行流程图

```
┌─────────────────────────────────────────────────────────────────────┐
│                      管线执行完整流程                                │
│                                                                     │
│   1️⃣ 创建管线                                                       │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   var pipeline = new AbilityPipeline<MyContext>();        │  │
│   │                                                             │  │
│   │   pipeline.AddPhase(new CastPhase());     // 引导阶段      │  │
│   │   pipeline.AddPhase(new ExecutePhase());   // 执行阶段      │  │
│   │   pipeline.AddPhase(new CooldownPhase());  // 冷却阶段      │  │
│   │                                                             │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                               │                                    │
│                               ▼                                    │
│   2️⃣ 启动管线                                                       │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   var context = new MyContext { Owner = this };           │  │
│   │   var run = pipeline.Start(config, context);               │  │
│   │                                                             │  │
│   │   // 内部执行：                                              │  │
│   │   // - 重置所有阶段                                          │  │
│   │   // - 创建 Run 实例                                         │  │
│   │   // - State → Executing                                    │  │
│   │   // - 触发 OnPipelineStart 事件                            │  │
│   │                                                             │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                               │                                    │
│                               ▼                                    │
│   3️⃣ 每帧驱动                                                       │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   void Update() {                                          │  │
│   │       run.Tick(Time.deltaTime);                            │  │
│   │   }                                                        │  │
│   │                                                             │  │
│   │   // Tick 内部：                                            │  │
│   │   // 1. 检查暂停/中断/取消                                  │  │
│   │   // 2. 如果当前阶段完成，进入下一阶段                       │  │
│   │   // 3. 如果是持续阶段，调用 OnUpdate                       │  │
│   │   // 4. 如果所有阶段完成，管线结束                           │  │
│   │                                                             │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                               │                                    │
│                               ▼                                    │
│   4️⃣ 阶段执行模式                                                   │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   瞬时阶段：                                                │  │
│   │   Execute() {                                               │  │
│   │       OnEnter();    // 初始化                               │  │
│   │       OnInstantExecute();  // 瞬时完成                      │  │
│   │       Complete();    // 标记完成                             │  │
│   │       OnExit();     // 清理                                 │  │
│   │   }                                                        │  │
│   │                                                             │  │
│   │   持续阶段：                                                │  │
│   │   Execute() { OnEnter(); }                                 │  │
│   │   OnUpdate(dt) {                                            │  │
│   │       _elapsed += dt;                                      │  │
│   │       if (_elapsed >= Duration) Complete();                │  │
│   │   }                                                        │  │
│   │                                                             │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                               │                                    │
│                               ▼                                    │
│   5️⃣ 管线结束                                                       │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   // 所有阶段完成或被中断                                   │  │
│   │   run.Finished += (state) => {                           │  │
│   │       if (state == EAbilityPipelineState.Completed) {     │  │
│   │           // 技能执行成功                                   │  │
│   │       }                                                    │  │
│   │   };                                                        │  │
│   │                                                             │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 四、核心组件详解

### 4.1 AbilityPipeline - 管线容器

**通俗解释**：Pipeline 是生产线的"蓝图"，定义了有哪些工位、按什么顺序执行。

```
┌─────────────────────────────────────────────────────────────┐
│                    AbilityPipeline 结构                       │
│                                                             │
│   public class AbilityPipeline<TCtx>                       │
│       where TCtx : IAbilityPipelineContext                  │
│   {                                                         │
│       // 事件通知                                           │
│       public AbilityPipelineEvents<TCtx> Events { get; }   │
│                                                             │
│       // 启动管线，返回执行实例                             │
│       public Run Start(config, context)                     │
│                                                             │
│       // 添加阶段                                           │
│       public void AddPhase(phase)                          │
│       public void InsertPhase(index, phase)                 │
│       public void RemovePhase(phaseId)                      │
│                                                             │
│       // 重置                                               │
│       public void Reset()                                   │
│   }                                                         │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   典型用法：                                          │  │
│   │                                                       │  │
│   │   var pipeline = new AbilityPipeline<MyContext>();   │  │
│   │   pipeline.AddPhase(new CastPhase());                 │  │
│   │   pipeline.AddPhase(new ExecutePhase());              │  │
│   │   pipeline.AddPhase(new CooldownPhase());             │  │
│   │                                                       │  │
│   │   var run = pipeline.Start(config, context);         │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 Run - 执行实例

**通俗解释**：Run 是生产线的"一次运行"，控制整个执行过程。

```
┌─────────────────────────────────────────────────────────────┐
│                    Run 执行状态机                             │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   Ready                                              │  │
│   │     │                                                │  │
│   │     │ Start()                                        │  │
│   │     ▼                                                │  │
│   │   Executing ◄─────────────────────────┐              │  │
```

```csharp
// 示例：引导吟唱
public class ChannelPhase : AbilityDurationalPhaseBase<MyContext>

    {
        Duration = 2.0f;  // 2秒引导

        ctx.Owner.ShowChannelEffect();
```

```
┌─────────────────────────────────────────────────────────────┐
│                    组合阶段一览                              │
│                                                             │
│   ┌───────────────────┬─────────────────────────────────┐ │
│   │ 阶段               │ 行为                             │ │

│   │ SequencePhase     │ 顺序执行子阶段，全部完成才算完成 │ │
```

#### SequencePhase - 顺序阶段

```csharp
// 示例：技能连招
var comboSequence = new AbilitySequencePhase<MyContext>();

comboSequence.AddSubPhase(new SecondHitPhase());
comboSequence.AddSubPhase(new ThirdHitPhase());
```

#### ParallelPhase - 并行阶段

```csharp
// 示例：同时播放特效、音效，造成伤害
var effectParallel = new AbilityParallelPhase<MyContext>();
effectParallel.AddSubPhase(new PlayVFXPhase("fireball_vfx"));
```

#### ConditionalPhase - 条件阶段

```csharp
// 示例：根据目标选择不同效果
var conditional = new AbilityConditionalPhase<MyContext>();

// 分支1：敌人存在 → 普通攻击
conditional.AddBranch(
    new LambdaCondition(ctx => ctx.Target != null && ctx.Target.IsAlive),
    new AttackPhase()
);

// 分支2：敌人不存在 → 冲锋接近
conditional.AddBranch(
    new LambdaCondition(ctx => ctx.Target == null || !ctx.Target.IsAlive),
    new DashPhase()
);

// 没有匹配条件时的行为
conditional.NoConditionBehavior = ENoConditionBehavior.Wait;  // 等待
```

## 六、使用指南

### 6.1 基础技能示例

```csharp
// 场景：实现一个"火球术"技能

public class FireballAbility : MonoBehaviour
{
    public float Damage = 100f;
    public float CastTime = 1.5f;
    public float Cooldown = 5f;
    public float Range = 10f;

    private AbilityPipeline<AbilityContext> _pipeline;
    private AbilityPipelineRun<AbilityContext> _currentRun;

    void Start()
    {
        // 构建管线
        _pipeline = new AbilityPipeline<AbilityContext>();

        // 添加阶段：引导 → 检测 → 施放 → 冷却
        _pipeline.AddPhase(new CastPhase(CastTime));
        _pipeline.AddPhase(new RangeCheckPhase(Range));
        _pipeline.AddPhase(new FireballExecutePhase(Damage));
        _pipeline.AddPhase(new CooldownPhase(Cooldown));
    }

    public bool TryCast(AbilityContext context)
    {
        if (_currentRun?.State == EAbilityPipelineState.Executing)
            return false;

        context.Owner = this;
        _currentRun = _pipeline.Start(null, context);
        return true;
    }

    void Update()
    {
        _currentRun?.Tick(Time.deltaTime);
    }
}

// 引导阶段
public class CastPhase : AbilityDurationalPhaseBase<AbilityContext>
{
    public CastPhase(float duration) : base("Cast")
    {
        Duration = duration;
    }

    protected override void OnEnter(AbilityContext ctx)
    {
        ctx.Owner.Animator.SetBool("isCasting", true);
        ctx.Owner.ShowCastEffect(ctx.TargetPosition);
    }

    protected override void OnTick(AbilityContext ctx, float dt)
    {
        // 被打断检查
        if (ctx.Owner.IsStunned || ctx.Owner.IsSilenced)
        {
            Interrupt();
        }
    }

    protected override void OnExit(AbilityContext ctx)
    {
        ctx.Owner.Animator.SetBool("isCasting", false);
        ctx.Owner.HideCastEffect();
    }

    protected override void OnInterrupt(AbilityContext ctx)
    {
        Debug.Log("施法被打断！");
    }
}

// 范围检测阶段
public class RangeCheckPhase : AbilityInstantPhaseBase<AbilityContext>
{
    private readonly float _range;

    public RangeCheckPhase(float range) : base("RangeCheck")
    {
        _range = range;
    }

    protected override void OnInstantExecute(AbilityContext ctx)
    {
        var distance = Vector3.Distance(ctx.Owner.transform.position, ctx.TargetPosition);
        if (distance > _range)
        {
            ctx.SetData("out_of_range", true);
            Debug.Log("目标超出范围！");
        }
    }
}

// 施放阶段
public class FireballExecutePhase : AbilityInstantPhaseBase<AbilityContext>
{
    private readonly float _damage;

    public FireballExecutePhase(float damage) : base("FireballExecute")
    {
        _damage = damage;
    }

    protected override void OnInstantExecute(AbilityContext ctx)
    {
        // 并行执行：生成火球 + 播放音效
        var effectPhase = new AbilityParallelPhase<AbilityContext>();
        effectPhase.AddSubPhase(new SpawnProjectilePhase("Fireball", ctx.TargetPosition));
        effectPhase.AddSubPhase(new PlaySoundPhase("fireball_cast"));

        // 计算伤害
        var projectile = ctx.GetData<Projectile>("projectile");
        projectile.OnHit += (target) =>
        {
            target.TakeDamage(_damage);
        };

        // 设置冷却
        ctx.Owner.CooldownManager.StartCooldown("Fireball", 5f);
    }
}

// 冷却阶段
public class CooldownPhase : AbilityDurationalPhaseBase<AbilityContext>
{
    public CooldownPhase(float cooldown) : base("Cooldown")
    {
        Duration = cooldown;
    }

    protected override void OnEnter(AbilityContext ctx)
    {
        ctx.Owner.SetCooldownActive(true);
    }

    protected override void OnExit(AbilityContext ctx)
    {
        ctx.Owner.SetCooldownActive(false);
    }
}
```

### 6.2 条件分支技能示例

```csharp
// 场景：实现一个"根据血量选择策略"的技能

public class TacticalAbilityPhase : AbilitySequencePhase<AbilityContext>
{
    public TacticalAbilityPhase()
    {
        // 第一步：判断血量
        AddSubPhase(new EvaluateHealthPhase());

        // 第二步：根据血量选择策略
        AddSubPhase(new ConditionalPhase<AbilityContext>(

public class EvaluateHealthPhase : AbilityInstantPhaseBase<AbilityContext>

    protected override void OnInstantExecute(AbilityContext ctx)

        ctx.SetData("health_percent", healthPercent);
    }
}

public class HealthBelowCondition : AbilityConditionNodeBase

    {
        _threshold = threshold;
    }

    public override bool Evaluate(IAbilityPipelineContext context)
    {
        if (context is IAbilityPipelineContext<AbilityContext> ctx)
        {
            var healthPercent = ctx.AbilityContext.GetData<float>("health_percent");
            return healthPercent < _threshold;
        }
        return false;
    }
```

### 6.3 组合技能示例

```csharp
// 场景：实现一个"连锁闪电"技能，同时造成伤害并附带效果

public class ChainLightningPipeline : AbilityPipeline<AbilityContext>
{
    public ChainLightningPipeline()
    {
        // 施法
        AddPhase(new CastPhase(0.5f));

        // 并行：播放特效 + 生成闪电链
        AddPhase(new AbilityParallelPhase<AbilityContext>
        {
            new PlayVFXPhase("lightning_vfx"),
            new ChainLightningExecutePhase(maxTargets: 5, damage: 50f, range: 5f)
        });

        // 冷却
        AddPhase(new CooldownPhase(8f));
    }
}

public class ChainLightningExecutePhase : AbilityInstantPhaseBase<AbilityContext>
{
    private readonly int _maxTargets;
    private readonly float _damage;
    private readonly float _range;

    public ChainLightningExecutePhase(int maxTargets, float damage, float range)
    {
        _maxTargets = maxTargets;
        _damage = damage;
        _range = range;
    }

    protected override void OnInstantExecute(AbilityContext ctx)
    {
        var targets = new List<Enemy>();
        var currentTarget = ctx.Target as Enemy;

        // 寻找连锁目标
        for (int i = 0; i < _maxTargets; i++)
        {
            if (currentTarget == null) break;

            targets.Add(currentTarget);
            currentTarget.TakeDamage(_damage);

            // 寻找下一个最近目标
            currentTarget = FindNearestEnemy(currentTarget.transform.position, _range);
        }

        ctx.SetData("chain_targets", targets);
    }
}
```

## 七、设计模式总结

### 7.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Template Method Pattern（模板方法）                    │
│      ─────────────────────────────────────                  │
│      Phase 基类定义生命周期骨架                              │
│      子类只实现具体逻辑                                      │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │  Execute() {                                     │   │
│      │      OnEnter();                                  │   │
│      │      OnExecute();  ← 子类实现                    │   │
│      │      OnExit();                                   │   │
│      │  }                                               │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Composite Pattern（组合模式）                          │
│      ─────────────────────────────────────                  │
│      Sequence、Parallel、Conditional 都是组合节点             │
│      可以无限嵌套                                            │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │  CompositePhase                                   │   │
│      │  ├─► PhaseA                                     │   │
│      │  ├─► CompositePhase                              │   │
│      │  │     ├─► PhaseB                               │   │
│      │  │     └─► PhaseC                               │   │
│      │  └─► PhaseD                                     │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Strategy Pattern（策略模式）                          │
│      ─────────────────────────────────────                  │
│      条件节点可以是不同策略实现                              │
│      And、Or、Not、Lambda                                   │
│                                                             │
│   4️⃣ State Pattern（状态模式）                              │
│      ─────────────────────────────────────                  │
│      Run 维护状态机的转换                                    │
│      Ready → Executing → Completed                          │
│                                                             │
│   5️⃣ Observer Pattern（观察者模式）                        │
│      ─────────────────────────────────────                  │
│      Events 通知各个观察者                                   │
│      OnPhaseStart、OnPhaseComplete、OnPipelineComplete      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 各模式解决的问题

| 模式 | 解决的问题 |
|------|------------|
| Template Method | 定义固定的生命周期，子类实现具体逻辑 |
| Composite | 用简单阶段组合复杂阶段 |
| Strategy | 不同的条件判断逻辑可替换 |
| State | 管理管线执行的不同状态 |
| Observer | 解耦事件通知，不用轮询 |

## 八、解决的问题与适用场景

### 8.1 Pipeline 模块解决什么问题

```
┌─────────────────────────────────────────────────────────────┐
│                    Pipeline 模块解决的问题                   │
│                                                             │
│   1️⃣ 阶段隔离                                            │
│      ─────────────────────────────────────                  │
│      每个阶段独立，不会互相干扰                              │
│      "引导"阶段只管引导，"施放"阶段只管施放              │
│                                                             │
│   2️⃣ 可中断处理                                           │
│      ─────────────────────────────────────                  │
│      支持Interrupt()，被打断时优雅清理                      │
│      "引导被打断" → 停止特效 → 恢复状态                    │
│                                                             │
│   3️⃣ 条件分支                                            │
│      ─────────────────────────────────────                  │
│      根据运行时条件选择不同执行路径                          │
│      "血量<30% → 撤退"  vs  "血量>=30% → 进攻"           │
│                                                             │
│   4️⃣ 并行执行                                            │
│      ─────────────────────────────────────                  │
│      特效、音效、伤害同时播放                                │
│      不需要嵌套回调                                         │
│                                                             │
│   5️⃣ 状态追踪                                            │
│      ─────────────────────────────────────                  │
│      每个阶段开始/完成都有事件通知                          │
│      调试时知道当前在哪一步                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 8.2 适用场景

```
┌─────────────────────────────────────────────────────────────┐
│                    适用场景                                  │
│                                                             │
│   ✅ 非常适合：                                             │
│   ──────────────────────────────────────────────────────── │
│   - MOBA 技能系统（引导、施放、冷却）                      │
│   - RPG 技能树（前置技能、条件解锁）                        │
│   - Buff/Debuff 系统（持续时间、叠加规则）                  │
│   - 复杂动画序列（动画 → 音效 → 特效 → 伤害）             │
│   - 对话/剧情系统（顺序展示、条件分支）                     │
│                                                             │
│   ❌ 不太适合：                                             │
│   ──────────────────────────────────────────────────────── │
│   - 需要逐帧精确控制的物理模拟                              │
│   - 简单的状态切换（用普通状态机即可）                      │
│   - 实时策略游戏的大规模单位控制                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
