# Ability-Kit 强类型触发器模块开发设计文档

> **阅读对象**：首次接触 Ability-Kit 触发器模块的开发者
>
> **文档目标**：让你理解触发器模块"是什么"、"解决了什么问题"、"为什么这样设计"、"怎么使用"
>
> **模块定位**：本模块是 Ability-Kit 框架中实现事件驱动逻辑的核心模块，采用强类型设计确保类型安全和编译期检查。

## 一、设计理念：为什么需要强类型触发器？

### 1.1 传统事件系统的问题

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     传统事件系统的问题                                   │
│                                                                         │
│   1️⃣ 类型不安全                                                        │
│   ──────────────────────────────────────                               │
│   // 用字符串或整数标识事件类型，容易写错                               │
│   void OnEvent(string eventType, object data) {                         │
│       if (eventType == "damage") {  // 容易写错！                      │
│           var d = (DamageData)data;                                    │
│           ...                                                          │
│       }                                                                │
│   }                                                                    │
│                                                                         │
│   2️⃣ 参数传递混乱                                                      │
│   ──────────────────────────────────────                               │
│   // object 类型需要到处 cast，容易出错                                  │
│   void OnEvent(object sender, object args) {                           │
│       var damage = (DamageEvent)args;  // 可能 cast 失败               │
│       var amount = (int)args.GetType().GetField("Amount").GetValue(args);  // 反射慢且危险 │
│   }                                                                    │
│                                                                         │
│   3️⃣ 条件逻辑散落各处                                                  │
│   ──────────────────────────────────────                               │
│   // 伤害事件来了，要做很多事：                                         │
│   void OnDamage(DamageEvent e) {                                       │
│       if (e.Amount > 10) ShowDamageNumber();                           │
│       if (e.Target.HP < 50) PlayLowHpEffect();                         │
│       if (e.Source.IsHero && e.Target.IsMinion) RecordKill();         │
│       if (e.Target.IsDead) DropLoot();                                  │
│   }                                                                    │
│   // 改动一个逻辑，要改这个方法，越来越长                               │
│                                                                         │
│   4️⃣ 难以复用和组合                                                    │
│   ──────────────────────────────────────                               │
│   // "伤害超过10"这个条件到处写                                         │
│   void OnSkill1() { if (damage > 10) ... }                            │
│   void OnSkill2() { if (damage > 10) ... }                            │
│   // 改一个地方要改三处                                                 │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.3 核心理念

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      触发器模块的四大承诺                                │
│                                                                         │
│   1️⃣ 类型安全                                                          │
│      泛型约束确保编译期检查                                              │

│                                                                         │

```

## 二、核心概念：从零理解强类型触发器

### 2.1 什么是"触发器（Trigger）"？

**通俗解释**：Trigger 是"规则"，当事件发生且满足条件时，就执行相应的动作。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      触发器 = 规则                                       │
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐   │
│   │                                                                 │   │
│   │   触发器 = 事件(泛型) + 条件 + 动作                              │   │
│   │                                                                 │   │
│   │   ┌─────────────────────────────────────────────────────────┐ │   │
│   │   │  当 (事件)     DamageEvent<伤害事件>                    │ │   │
│   │   │  如果 (条件)   Amount > 10                              │ │   │
│   │   │  那么 (动作)   ShowDamageNumber()                       │ │   │
│   │   └─────────────────────────────────────────────────────────┘ │   │
│   │                                                                 │   │
│   │   强类型优势：                                                  │   │
│   │   - DamageEvent.Amount 是 double 类型，编译期已知             │   │
│   │   - 无需反射或装箱拆箱                                          │   │
│   │   - 条件表达式直接访问事件字段                                  │   │
│   │                                                                 │   │
│   └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 代码位置 |
|------|----------|----------|
| **ITrigger** | 触发器接口，定义评估和执行 | `Runtime/Runtime/ITrigger.cs` |
| **EventKey** | 事件的类型安全标识 | `../AbilityKit.Core` |
| **TriggerRunner** | 触发器调度器，管理注册和执行 | `Runtime/Runtime/TriggerRunner.cs` |
| **ExecCtx** | 执行上下文，包含所有可用资源 | `Runtime/Runtime/ExecCtx.cs` |

| **Blackboard** | 共享数据存储 | `Runtime/Blackboard/IBlackboard.cs` |
| **FunctionRegistry** | 条件函数注册表 | `Runtime/Registry/FunctionRegistry.cs` |
| **ActionRegistry** | 动作函数注册表 | `Runtime/Registry/ActionRegistry.cs` |
| **TriggerPlan** | 触发器计划配置 | `Runtime/Plan/TriggerPlan.cs` |
| **ExecutionControl** | 执行控制，支持短路 | `Runtime/Runtime/ExecutionControl.cs` |

### 3.3 事件派发流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              事件派发流程                                    │
│                                                                             │
│   1️⃣ 派发强类型事件                                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   // 强类型事件派发                                                  │  │
│   │   var key = new EventKey<DamageEvent>(                              │  │
│   │       StableStringId.Get("event:damage"));                          │  │
│   │                                                                     │  │
│   │   eventBus.Publish(key, new DamageEvent {                           │  │
│   │       Amount = 50,                                                  │  │
│   │       TargetId = enemyId,                                           │  │
│   │       SourceId = heroId                                             │  │
│   │   });                                                                │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                      │                                     │
│                                      ▼                                     │
│   2️⃣ EventBus 路由到 Channel                                               │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   // EventBus 根据类型获取 EventChannel                             │  │
│   │   var dict = _channelsByArgsType[typeof(DamageEvent)];             │  │
│   │   var channel = dict[key];                                         │  │
│   │                                                                     │  │
│   │   // Queued 模式：加入队列                                          │  │
│   │   channel.Enqueue(args, null);                                     │  │
│   │                                                                     │  │
│   │   // Immediate 模式：直接分发                                       │  │
│   │   channel.DispatchImmediate(args, null);                           │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                      │                                     │
│                                      ▼                                     │
│   3️⃣ TriggerRunner 接收并调度                                              │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   // Dispatcher.OnEvent 回调被调用                                  │  │
│   │   _runner.Dispatch(_key, in args, control, _list);                │  │
│   │                                                                     │  │
│   │   ┌─────────────────────────────────────────────────────────────┐  │  │
│   │   │  获取该事件的触发器列表                                      │  │  │
│   │   │  var triggers = _handlers[eventKey];                       │  │  │
│   │   │                                                             │  │  │
│   │   │  // 已按 Phase → Priority → Order 排序                    │  │  │
│   │   └─────────────────────────────────────────────────────────────┘  │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                      │                                     │
│                                      ▼                                     │
│   4️⃣ 遍历触发器执行                                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                                                                     │  │
│   │   for (int i = 0; i < triggers.Count; i++) {                     │  │
│   │       var entry = triggers[i];                                     │  │
│   │       var ok = entry.Trigger.Evaluate(in args, in execCtx);      │  │
│   │                                                                     │  │
│   │       if (control.StopPropagation || control.Cancel) break;       │  │
│   │                                                                     │  │
│   │       if (ok) {                                                   │  │
│   │           entry.Trigger.Execute(in args, in execCtx);             │  │
│   │       }                                                            │  │
│   │   }                                                                │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────────────────────┐

│   ──────────────────────────────────────                                    │
│   TriggerRunner 订阅 EventBus 的事件派发                                   │
│   收到事件后通知所有注册的触发器                                            │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │                                                                       │   │

│   │              ┌────────────────────────────────────────────────┐    │   │
│   │              │  ITrigger[]                                      │    │   │

│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │                                                                       │   │
│   │   PredicateKind.None     ──► 永远通过                               │   │

│   │                                                                       │   │

│                                                                             │

│   │   ├─ ActionRegistry        (动作注册表)                              │   │

│   │   ├─ IPayloadAccessor     (Payload访问器)                          │   │

│   │   └─ ExecPolicy           (执行策略)                                 │   │
│   │                                                                       │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │

│   支持延迟解析和确定性检查                                                  │

│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 各模式解决的问题

| 模式 | 解决的问题 |
|------|------------|
| Observer | 事件派发和处理解耦 |
| Strategy | 条件判断逻辑可替换 |
| Dependency Injection | 依赖管理简单，测试方便 |
| Visitor | 按优先级有序执行 |
| Registry | ID 到 Delegate 的类型安全映射 |
| Interpreter | 表达式运行时求值 |

## 六、使用指南

### 6.1 基础使用示例

```csharp
// 场景：战斗伤害飘字系统

// 1. 定义强类型事件
public readonly struct DamageEvent
{
    public readonly double Amount;
    public readonly int TargetId;
    public readonly int SourceId;
    
    public DamageEvent(double amount, int targetId, int sourceId)
    {
        Amount = amount;
        TargetId = targetId;
        SourceId = sourceId;
    }
}

// 2. Payload 访问器（从事件中提取字段）
public sealed class DamageEventPayloadAccessor : IPayloadIntAccessor<DamageEvent>
{
    private static readonly int AmountFieldId = StableStringId.Get("payload:amount");
    
    public bool TryGet(in DamageEvent args, int fieldId, out int value)
    {
        if (fieldId == AmountFieldId)
        {
            value = (int)args.Amount;
            return true;
        }
        value = default;
        return false;
    }
}

// 3. 创建核心组件
var eventBus = new EventBus(EventBusOptions.Default with { DispatchMode = EEventDispatchMode.Queued });
var functionRegistry = new FunctionRegistry();
var actionRegistry = new ActionRegistry();
var blackboards = new DictionaryBlackboardResolver();
var payloads = new PayloadAccessorRegistry();
payloads.RegisterIntAccessor(new DamageEventPayloadAccessor());

// 4. 创建 Runner
var runner = new TriggerRunner<BattleContext>(
    eventBus,
    functionRegistry,
    actionRegistry,
    contextSource: null,
    observer: null,
    blackboards,
    payloads
);

// 5. 注册条件函数
var highDamagePredicateId = new FunctionId(StableStringId.Get("pred:high_damage"));
functionRegistry.Register(
    highDamagePredicateId,
    (PlannedTrigger<DamageEvent, BattleContext>.Predicate0)((evt, ctx) => evt.Amount > 10),
    isDeterministic: true);

// 6. 注册动作
var showDamageActionId = new ActionId(StableStringId.Get("action:show_damage"));
actionRegistry.Register(
    showDamageActionId,
    (PlannedTrigger<DamageEvent, BattleContext>.Action0)((evt, ctx) =>
    {
        // 通过 ctx.Context 访问上下文
        var pos = ctx.Context.GetTargetPosition(evt.TargetId);
        UIManager.ShowDamageNumber(pos, evt.Amount);
    }),
    isDeterministic: true);

// 7. 创建触发器
var trigger = new DelegateTrigger<DamageEvent, BattleContext>(
    predicate: (evt, ctx) => evt.Amount > 10,
    actions: (evt, ctx) => ShowDamage(evt.Amount));

// 8. 注册触发器
var damageKey = new EventKey<DamageEvent>(StableStringId.Get("event:damage"));
runner.Register(damageKey, trigger, phase: 0, priority: 100);

// 9. 派发事件
eventBus.Publish(damageKey, new DamageEvent(50, enemyId, heroId));
eventBus.Flush();
```

// 场景：使用 TriggerPlan 实现可配置的触发器

### 6.3 使用 RPN 表达式

```csharp
// 场景：复杂条件表达式

// 1. 创建布尔表达式
// 条件: (damage > 10) AND (target.health < 50)
var predicateExpr = new PredicateExprPlan(new[]
{
    // payload.amount > 10
    BoolExprNode.Compare(
        ECompareOp.GreaterThan,
        NumericValueRef.PayloadField(AmountFieldId),
        NumericValueRef.Const(10)),
    
    // bb.target.health < 50
    BoolExprNode.Compare(
        ECompareOp.LessThan,
        NumericValueRef.Blackboard(targetBoardId, healthKeyId),
        NumericValueRef.Const(50)),
    
    // AND
    BoolExprNode.And()
});

// 2. 使用表达式创建计划
var plan = new TriggerPlan<DamageEvent>(
    phase: 0,
    priority: 100,
    predicateExpr: predicateExpr,
    actions: new[] { new ActionCallPlan(triggerSkillActionId) });

// 3. 注册
runner.RegisterPlan(damageKey, plan);
```

### 6.4 确定性执行策略

```csharp
// 场景：帧同步战斗，需要确定性执行

// 1. 设置确定性策略
var policy = ExecPolicy.DeterministicOnly;  // RequireDeterministic = true

// 2. 创建 Runner 时传入策略
var runner = new TriggerRunner<BattleContext>(
    eventBus,
    functionRegistry,
    actionRegistry,
    policy: policy);

// 3. 注册时检查确定性
actionRegistry.Register(
    showDamageActionId,
    action,
    isDeterministic: true);  // 必须为 true，否则抛异常

// 4. 非确定性动作会被拒绝
try
{
    actionRegistry.Register(
        randomActionId,
        () => Random.value > 0.5,  // 非确定性
        isDeterministic: false);   // 设为 false
    
    // 在 DeterministicOnly 模式下，这会抛异常
}
catch (InvalidOperationException ex)
{
    // Non-deterministic action is not allowed by policy
}
```
