## 一、设计理念：为什么要做触发器模块？

### 1.1 传统做法的问题

```
❌ 传统做法的问题：

1. 事件处理散落各处
   ┌─────────────────────────────────────────────────────────┐
   │ // 伤害事件来了，要做很多事：                           │
   │ void OnDamage(DamageEvent e) {                        │
   │     if (e.Amount > 10) {                              │
   │         ShowDamageNumber();  // 飘字                 │
   │     }                                                 │
   │     if (e.Target.HP < 50) {                          │
   │         PlayLowHpEffect();    // 低血量特效          │
   │     }                                                 │
   │     if (e.Source.IsHero && e.Target.IsMinion) {      │
   │         StatManager.Record("hero_kill_minion");      │
   │     }                                                 │
   │     if (e.Target.IsDead) {                           │
   │         DropLoot();            // 掉落处理          │
   │     }                                                 │
   │ }                                                      │
   │                                                          │
   │ // 改动一个逻辑，要改这个方法                          │
   │ // 加新条件，要在这个方法里加 if                       │
   │ // 越来越长，越来越乱                                 │
   └─────────────────────────────────────────────────────────┘

2. 条件逻辑难以复用
   ┌─────────────────────────────────────────────────────────┐
   │ // "伤害超过10"这个条件到处写：                        │
   │ void OnSkill1() { if (damage > 10) ... }             │
   │ void OnSkill2() { if (damage > 10) ... }             │
   │ void OnSkill3() { if (damage > 10) ... }             │
   │                                                          │
   │ // 改一个地方要改三处，漏了就是Bug                    │
   └─────────────────────────────────────────────────────────┘

3. 事件和处理耦合
   ┌─────────────────────────────────────────────────────────┐
   │ // 派发事件的代码要知道谁在监听                        │
   │ // 监听者要知道事件是怎么派的                          │
   │ // 改事件结构，所有监听者都要改                        │
   └─────────────────────────────────────────────────────────┘

4. 难以配置和可视化
   ┌─────────────────────────────────────────────────────────┐
   │ // 想让策划改规则？                                    │
   │ // "要改代码..."                                      │
   │ // 想看哪些事件触发了什么？                            │
   │ // "要加日志..."                                      │
   └─────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解触发器

### 2.1 什么是"触发器（Trigger）"？

**通俗解释**：Trigger 是"规则"，当事件发生且满足条件时，就执行相应的动作。

```
┌─────────────────────────────────────────────────────────────┐
│                    触发器 = 规则                            │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   触发器 = 事件 + 条件 + 动作                        │  │
│   │                                                       │  │
│   │   ┌───────────────────────────────────────────┐    │  │
│   │   │  当 (事件)     伤害事件来了               │    │  │
│   │   │  如果 (条件)   伤害值 > 10                │    │  │
│   │   │  那么 (动作)   显示飘字                   │    │  │
│   │   └───────────────────────────────────────────┘    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   触发器示例：                                              │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   Trigger: 伤害飘字                                  │  │
│   │   Event:   OnDamage                                 │  │
│   │   Cond:    Amount > 10                               │  │
│   │   Action:  ShowDamageNumber()                       │  │
│   │                                                       │  │
│   │   Trigger: 低血量警告                                 │  │
│   │   Event:   OnHealthChanged                          │  │
│   │   Cond:    HP < MaxHP * 0.3                          │  │
│   │   Action:  PlayWarningEffect()                       │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **Event** | 发生的事情 | 警报响起 |
| **Trigger** | 规则：事件+条件+动作 | 应急预案 |
| **Condition** | 是否执行的判断 | 警报级别 > 3？ |
| **Action** | 满足条件后做的事 | 疏散人员 |
| **EventBus** | 事件派发总线 | 广播站 |
| **TriggerRunner** | 触发器调度器 | 应急指挥中心 |
| **ExecCtx** | 执行上下文，包含所有可用资源 | 应急手册 |
| **Blackboard** | 共享数据存储 | 信息板 |

### 3.2 事件派发流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                      事件派发流程                                    │
│                                                                     │
│   1️⃣ 派发事件                                                     │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   // 代码中派发事件                                          │  │
│   │   eventBus.Publish(damageKey, new DamageEvent {            │  │
│   │       Amount = 50,                                          │  │
│   │       Target = enemy,                                       │  │
│   │       Source = hero                                         │  │
│   │   });                                                       │  │
│   │                                                             │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                               │                                    │
│                               ▼                                    │
│   2️⃣ 事件通道处理                                                  │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   // Queued 模式：加入队列                                  │  │
│   │   eventChannel._queue.Enqueue(event);                      │  │
│   │                                                             │  │
│   │   // Immediate 模式：直接处理                                │  │
│   │   eventChannel.Handler(event);                              │  │
│   │                                                             │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                               │                                    │
│                               ▼                                    │
│   3️⃣ TriggerRunner 调度                                            │
│   ┌─────────────────────────────────────────────────────────────┐  │
│   │                                                             │  │
│   │   // 获取该事件的触发器列表                                  │  │
│   │   var triggers = _handlers[eventKey];                      │  │
│   │                                                             │  │
│   │   // 按优先级排序后遍历                                      │  │
│   │   foreach (var trigger in triggers) {                      │  │
│   │       if (trigger.Evaluate(event, ctx)) {                 │  │
│   │           trigger.Execute(event, ctx);                     │  │
│   │       }                                                     │  │
│   │   }                                                        │  │
│   │                                                             │  │
│   └─────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```

│                      条件求值流程                                    │

│                                                                     │
│   ┌─────────────────────────────────────────────────────────────┐  │

## 四、核心组件详解

### 4.1 EventBus - 事件总线

**通俗解释**：EventBus 是"广播站"，负责派发事件到各个监听者。

```
┌─────────────────────────────────────────────────────────────┐
│                    EventBus 结构                              │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   public sealed class EventBus : IEventBus           │  │
│   │   {                                                  │  │
│   │       // 派发模式                                    │  │
│   │       public enum EEventDispatchMode {              │  │
│   │           Queued = 0,     // 队列模式               │  │
│   │           Immediate = 1,  // 即时模式               │  │
│   │       }                                             │  │
│   │                                                       │  │
│   │       // 派发事件                                    │  │
│   │       public void Publish<T>(EventKey<T> key, T args)│  │
│   │                                                       │  │
│   │       // 刷新队列（Queued模式用）                    │  │
│   │       public void Flush()                            │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   派发模式对比：                                      │  │
│   │                                                       │  │
│   │   Queued（队列模式）                                  │  │
│   │   ─────────────────────────────────────────────     │  │
│   │   1. 事件放入队列                                     │  │
│   │   2. 一帧结束时 Flush()                              │  │
│   │   3. 批量处理所有队列中的事件                         │  │
│   │   → 防止死循环                                       │  │
│   │                                                       │  │
│   │   Immediate（即时模式）                                │  │
│   │   ─────────────────────────────────────────────     │  │
│   │   1. 立刻调用所有监听者                               │  │
│   │   → 性能更好，但可能死循环                           │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 TriggerRunner - 触发器调度器

**通俗解释**：TriggerRunner 是"应急指挥中心"，管理所有触发器并按规则调度执行。

```
┌─────────────────────────────────────────────────────────────┐
│                    TriggerRunner 结构                        │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   public sealed class TriggerRunner<TCtx>            │  │
│   │   {                                                  │  │
│   │       // 注册触发器                                   │  │
│   │       public IDisposable Register(...)               │  │
│   │                                                       │  │

## 五、设计模式总结

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Observer Pattern（观察者模式）                        │
│      ─────────────────────────────────────                  │
│      TriggerRunner 观察 EventBus 的事件派发                │
│      收到事件后通知所有注册的触发器                         │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   EventBus ──► Publish(event) ──► TriggerRunner │   │
│      │                                    │              │   │
│      │                                    ▼              │   │
│      │         ┌───────────────────────────────────┐  │   │
│      │         │  ITrigger[]                       │  │   │
│      │         │  ├─► Trigger A (观察者)          │  │   │
│      │         │  ├─► Trigger B (观察者)          │  │   │
│      │         │  └─► Trigger C (观察者)          │  │   │
│      │         └───────────────────────────────────┘  │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Strategy Pattern（策略模式）                          │
│      ─────────────────────────────────────                  │
│      PredicateKind 支持三种条件策略                         │
│      None / Function / Expr 可以自由切换                    │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   PredicateKind.None     ──► 永远通过          │   │
│      │   PredicateKind.Function ──► 委托函数策略     │   │
│      │   PredicateKind.Expr     ──► RPN表达式策略   │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Dependency Injection（依赖注入）                       │
│      ─────────────────────────────────────                  │
│      ExecCtx 注入所有依赖，触发器不需要自己找依赖           │
│                                                             │
│   4️⃣ Visitor Pattern（访问者模式）                        │
│      ─────────────────────────────────────                  │
│      TriggerRunner 按 Phase→Priority 顺序遍历触发器         │
│      ExecCtx 提供统一的资源访问接口                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 各模式解决的问题

| 模式 | 解决的问题 |
|------|------------|
| Observer | 事件派发和处理解耦 |
| Strategy | 条件判断逻辑可替换 |
| Dependency Injection | 依赖管理简单，测试方便 |
| Visitor | 按优先级有序执行 |

### 6.1 基础使用示例

```csharp
// 场景：战斗伤害飘字系统

// 1. 定义事件
public readonly struct DamageEvent
{
    public readonly double Amount;
    public readonly int TargetId;
    public readonly int SourceId;
}

// 2. 创建核心组件
var eventBus = new EventBus();
var functionRegistry = new FunctionRegistry();
var actionRegistry = new ActionRegistry();
var blackboards = new DictionaryBlackboardResolver();

// 3. 创建 Runner
var runner = new TriggerRunner<TriggerContext>(
    eventBus,
    functionRegistry,
    actionRegistry,
    blackboards
);

// 4. 注册条件函数
var highDamagePredicateId = new FunctionId(
    StableStringId.Get("pred:high_damage"));

functionRegistry.Register<PlannedTrigger<DamageEvent, TriggerContext>.Predicate0>(
    highDamagePredicateId,
    (evt, ctx) => evt.Amount > 10,
    isDeterministic: true);

// 5. 注册动作
var showDamageActionId = new ActionId(
    StableStringId.Get("action:show_damage"));

actionRegistry.Register<PlannedTrigger<DamageEvent, TriggerContext>.Action0>(
    showDamageActionId,
    (evt, ctx) =>
    {
        var pos = ctx.Context.GetTargetPosition(evt.TargetId);
        UIManager.ShowDamageNumber(pos, evt.Amount);
    },
    isDeterministic: true);

// 6. 创建并注册计划
var plan = new TriggerPlan<DamageEvent>(
    phase: 0,
    priority: 100,
    predicateId: highDamagePredicateId,
    actions: new[] { new ActionCallPlan(showDamageActionId) }
);

var damageKey = new EventKey<DamageEvent>(
    StableStringId.Get("event:damage"));
runner.RegisterPlan(damageKey, plan);

// 7. 派发事件
eventBus.Publish(damageKey, new DamageEvent
{
    Amount = 50,
    TargetId = enemyId,
    SourceId = heroId
});
eventBus.Flush();
```

### 6.3 完整示例：战斗事件系统

```csharp
public class BattleEventSystem : IDisposable
{
    private readonly EventBus _eventBus;
    private readonly TriggerRunner<BattleCtx> _runner;
    private readonly DictionaryBlackboardResolver _blackboards;

    public BattleEventSystem(BattleCtx ctx)
    {
        _eventBus = new EventBus();
        _blackboards = new DictionaryBlackboardResolver();

        // 创建战斗黑板
        var battleBB = new DictionaryBlackboard();
        _blackboards.Register(StableStringId.Get("bb:battle"), battleBB);

        _runner = new TriggerRunner<BattleCtx>(
            _eventBus,
            new FunctionRegistry(),
            new ActionRegistry(),
            _blackboards,
            contextSource: new BattleContextSource(ctx)
        );

        // 注册触发器
        RegisterDamageTriggers();
        RegisterDeathTriggers();
        RegisterKillTriggers();
    }

    private void RegisterDamageTriggers()
    {
        // 高伤害触发飘字
        var plan = new TriggerPlan<DamageEvent>(
            phase: 0,
            priority: 100,
            predicateId: FunctionId.HighDamage,
            actions: new[] { ActionId.ShowDamageNumber }
        );
        _runner.RegisterPlan(EventKeys.Damage, plan);

        // 致命伤害触发死亡
        var lethalPlan = new TriggerPlan<DamageEvent>(
            phase: 1,
            priority: 100,
            predicateId: FunctionId.LethalDamage,
            actions: new[] { ActionId.TriggerDeath }
        );
        _runner.RegisterPlan(EventKeys.Damage, lethalPlan);
    }

    public void PublishDamage(double amount, int target, int source)
    {
        _eventBus.Publish(EventKeys.Damage, new DamageEvent
        {
            Amount = amount,
            TargetId = target,
            SourceId = source
        });
        _eventBus.Flush();
    }

    public void Dispose()
    {
        _runner.Dispose();
    }
}
```
