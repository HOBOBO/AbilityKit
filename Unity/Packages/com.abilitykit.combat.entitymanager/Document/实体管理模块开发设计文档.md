# Ability-Kit 实体管理模块开发设计文档

> **阅读对象**：首次接触 Ability-Kit 实体管理模块的开发者
>
> **文档目标**：让你理解实体管理模块"是什么"、"解决了什么问题"、"为什么这样设计"、"怎么使用"

## 一、设计理念：为什么要做实体管理模块？

### 1.1 传统游戏中的痛点

```
❌ 传统做法的问题：

1. 实体查询效率低下
   ┌─────────────────────────────────────────────────────────┐
   │ // 需要找到"所有蓝队英雄"                               │
   │ List<Unit> GetBlueTeamHeroes() {                        │
   │     var result = new List<Unit>();                     │
   │     foreach (var unit in allUnits) {                   │
   │         if (unit.Team == Team.Blue &&                  │
   │             unit.Type == UnitType.Hero) {              │
   │             result.Add(unit);                         │
   │         }                                              │
   │     }                                                  │
   │     return result;                                     │
   │ }                                                      │
   │                                                          │
   │ // 1000个单位 × 10种查询 = 每次都是O(N)遍历           │
   │ // 帧同步游戏，每帧都要查，CPU爆炸                      │
   └─────────────────────────────────────────────────────────┘

2. 查询条件分散
   ┌─────────────────────────────────────────────────────────┐
   │ // 蓝队英雄要在TeamManager里查                          │
   │ // 技能目标要在SkillSystem里查                          │
   │ // 视野单位要在VisionSystem里查                          │
   │ // Buff目标要在BuffSystem里查                            │
   │ // 代码散落各处，不知道去哪找                            │
   └─────────────────────────────────────────────────────────┘

3. 索引逻辑重复
   ┌─────────────────────────────────────────────────────────┐
   │ // TeamManager: if (unit.Team == team) ...            │
   │ // SkillSystem: if (unit.Team == team) ...            │
   │ // BuffSystem:  if (unit.Team == team) ...            │
   │ // VisionSystem: if (unit.Team == team) ...            │
   │ // 同样的逻辑写N遍，改一个要改N处                       │
   └─────────────────────────────────────────────────────────┘

4. 实体属性变化后索引不同步
   ┌─────────────────────────────────────────────────────────┐
   │ // 单位转阵营了                                         │
   │ unit.Team = Team.Red;  // ← 蓝队列表不会自动更新！    │
   │ // 继续在蓝队列表里找到它，查了半天发现它是红的       │
   └─────────────────────────────────────────────────────────┘
```

### 1.2 实体管理模块的解决方案

```
✅ 实体管理的设计思路：

┌─────────────────────────────────────────────────────────────┐
│                       核心思想                              │
│                                                             │
│   用"索引表"预先组织实体，查询时直接查表                    │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   注册实体时：                                        │  │
│   │   ┌───────────────────────────────────────────┐    │  │
│   │   │  unit.Team = Blue;                        │    │  │
│   │   │  unit.Type = Hero;                        │    │  │

│   │   │  ├─► 蓝队列表: [unit1, unit2, ...]     │    │  │

│   │   │  └─► 单位列表: [unit1, unit2, ...]     │    │  │
│   │   └───────────────────────────────────────────┘    │  │

│   │   └───────────────────────────────────────────┘    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                    实体管理模块的三大承诺                    │
│                                                             │

```

## 二、核心概念：从零理解实体管理

### 2.1 什么是"索引（Index）"？

**通俗解释**：索引就像图书馆的"分类目录"，告诉你"蓝队英雄在第3排第5格"。

```
┌─────────────────────────────────────────────────────────────┐
│                    索引 = 分类目录                          │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   图书馆目录：                                        │  │
│   │   ┌─────────────────────────────────────────────┐   │  │
│   │   │                                               │   │  │
│   │   │  按队伍分类：                                 │   │  │
│   │   │  ├─► 蓝队: [英雄1, 小兵1, 建筑1, ...]    │   │  │
│   │   │  └─► 红队: [英雄2, 小兵2, 建筑2, ...]    │   │  │
│   │   │                                               │   │  │
│   │   │  按类型分类：                                 │   │  │
│   │   │  ├─► 英雄: [英雄1, 英雄2, ...]           │   │  │
│   │   │  ├─► 小兵: [小兵1, 小兵2, ...]           │   │  │
│   │   │  └─► 建筑: [建筑1, 建筑2, ...]           │   │  │
│   │   │                                               │   │  │
│   │   │  按主人分类：                                 │   │  │
│   │   │  ├─► 玩家1: [英雄1, 小兵1_1, ...]         │   │  │
│   │   │  └─► 玩家2: [英雄2, 小兵2_1, ...]         │   │  │
│   │   │                                               │   │  │
│   │   └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   查询"蓝队英雄"：                                          │
│   byTeam.Get(Team.Blue) ∩ byType.Get(UnitType.Hero)       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **EntityRegistry** | 实体注册表，索引的管理者 | 图书馆管理员 |
| **BattleEntityManager** | 对外接口，管理实体的入口 | 图书馆前台 |
| **EntityIndex** | 索引表，按某个维度组织实体 | 分类目录 |
| **KeyedEntityIndex** | 单键索引，一个实体只有一个键 | "按队伍分类"（每个单位只属于一队） |
| **MultiKeyEntityIndex** | 多键索引，一个实体可以有多个键 | "按Buff分类"（一个单位可以有多个Buff） |
| **EntityUpdate** | 实体更新事件 | 属性变更通知 |

## 四、核心组件详解

### 4.1 BattleEntityManager - 对外接口

**通俗解释**：Manager 是"图书馆前台"，所有操作都通过它。

```
┌─────────────────────────────────────────────────────────────┐
│                    BattleEntityManager 接口                  │
│                                                             │
│   public sealed class BattleEntityManager<TId>              │
│   {                                                         │
│       public readonly EntityRegistry<TId> Registry;        │
│                                                             │
│       // 实体管理                                            │
│       public void Add(TId id);                             │
│       public bool Remove(TId id);                           │
│       public bool Contains(TId id);                         │
│       public void AddRange(IEnumerable<TId> ids);           │
│       public int RemoveRange(IEnumerable<TId> ids);         │
│                                                             │
│       // 索引创建                                            │
│       public KeyedEntityIndex<TKey, TId>                    │
│           CreateKeyedIndex<TKey>(int updateType, ...);     │
│       public MultiKeyEntityIndex<TKey, TId>                 │
│           CreateMultiKeyIndex<TKey>(...);                   │
│                                                             │
│       // 通知更新                                            │
│       public void NotifyUpdated(TId id, EntityUpdate u);   │
│       public int NotifyUpdatedBatch(...);                    │
│   }                                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 EntityRegistry - 核心注册表

**通俗解释**：Registry 是"图书馆管理员"，管理所有实体和索引。

```
┌─────────────────────────────────────────────────────────────┐
│                    EntityRegistry 结构                       │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   private readonly HashSet<TId> _entities;          │  │
│   │   private readonly List<IEntityIndex<TId>> _indices;│  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   AddIndex(index)                                     │  │
│   │   ─────────────────                                   │  │
│   │   1. _indices.Add(index)                             │  │
│   │   2. 遍历 _entities，retroactively populate index    │  │
│   │      // 新索引自动包含所有已有实体                   │  │
│   │                                                       │  │
│   │   Add(id)                                             │  │
│   │   ─────────                                           │  │
│   │   1. _entities.Add(id)                               │  │
│   │   2. 遍历 _indices，index.OnEntityAdded(id)          │  │
│   │                                                       │  │
│   │   Remove(id)                                          │  │
│   │   ──────────                                          │  │
│   │   1. _entities.Remove(id)                             │  │
│   │   2. 遍历 _indices，index.OnEntityRemoved(id)         │  │
│   │                                                       │  │
│   │   NotifyUpdated(id, update)                            │  │
│   │   ───────────────────────                              │  │
│   │   1. if (!_entities.Contains(id)) return;             │  │
│   │   2. 遍历 _indices                                    │  │
│   │   3. 跳过不关心此类型的索引                          │  │
│   │   4. index.OnEntityUpdated(id, update)                │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 KeyedEntityIndex - 单键索引

**通俗解释**：KeyedIndex 是"单标签分类目录"，每个实体只有一个标签。

```
┌─────────────────────────────────────────────────────────────┐
│                    KeyedEntityIndex 结构                     │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   private readonly Dictionary<TKey, HashSet<TId>>    │  │
│   │       _map;  // 键 → 实体集合                        │  │
│   │                                                       │  │
│   │   private readonly Dictionary<TId, TKey>             │  │
│   │       _currentKey;  // 实体 → 当前键                 │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   SetKey(id, key)                                     │  │
│   │   ─────────────                                       │  │
│   │   1. 如果实体已有键：                                 │  │
│   │      _map[oldKey].Remove(id);  // 从旧键移除         │  │
│   │   2. _currentKey[id] = key;                          │  │
│   │   3. _map[key].Add(id);  // 添加到新键             │  │
│   │                                                       │  │
│   │   Get(key)                                            │  │
│   │   ──────                                              │  │
│   │   return _map.TryGetValue(key, out var set) ? set :  │  │
│   │          EmptyCollection;  // O(1) 字典查找          │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   使用场景：                                                │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  - 队伍索引：每个单位只属于一个队伍                  │  │
│   │  - 类型索引：每个单位只有一个主类型                  │  │
│   │  - 所有者索引：每个单位只属于一个玩家                │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.4 MultiKeyEntityIndex - 多键索引

**通俗解释**：MultiKeyIndex 是"多标签分类目录"，一个实体可以有多个标签。

```
┌─────────────────────────────────────────────────────────────┐
│                    MultiKeyEntityIndex 结构                  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   private readonly Dictionary<TKey, HashSet<TId>>    │  │
│   │       _map;  // 键 → 实体集合                       │  │
│   │                                                       │  │
│   │   private readonly Dictionary<TId, HashSet<TKey>>   │  │
│   │       _keysByEntity;  // 实体 → 键集合（双向映射）  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   AddKey(id, key)                                    │  │
│   │   ──────────────                                      │  │
│   │   1. _map[key].Add(id);  // 键 → 实体              │  │
│   │   2. _keysByEntity[id].Add(key);  // 实体 → 键     │  │
│   │                                                       │  │
│   │   RemoveKey(id, key)                                  │  │
│   │   ─────────────────                                    │  │
│   │   1. _map[key].Remove(id);                           │  │
│   │   2. _keysByEntity[id].Remove(key);                  │  │
│   │                                                       │  │
│   │   Get(key)                                            │  │
│   │   ──────                                              │  │
│   │   return _map.TryGetValue(key, out var set) ? set :  │  │
│   │          EmptyCollection;                             │  │
│   │                                                       │  │
│   │   TryGetKeys(id, out keys)                            │  │
│   │   ────────────────────────                            │  │
│   │   // 查询某实体有哪些键（MultiKey独有）              │  │
│   │   return _keysByEntity.TryGetValue(id, out var set);  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   使用场景：                                                │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  - Buff索引：一个单位可以有多个Buff标签              │  │
│   │  - 状态索引：一个单位可以有多个状态标签              │  │
│   │  - 特效索引：一个单位可以有多个特效标签              │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 五、设计模式总结

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Mediator Pattern（中介者模式）                        │
│      ─────────────────────────────────────                  │
│      EntityRegistry 作为中介                                  │
│      实体和索引不直接通信，都通过 Registry 中转              │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   实体 ──► Registry ──► 索引                    │   │
│      │             ▲                                    │   │
│      │             │                                    │   │
│      │           索引 ◄── Registry ◄── 实体            │   │
│      │                                                   │   │
│      │   实体和索引互不认识，只认识 Registry            │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Observer Pattern（观察者模式）                        │
│      ─────────────────────────────────────                  │
│      索引观察 Registry 的实体生命周期事件                    │
│      收到通知后更新自己的内部状态                            │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │  IEntityIndex                                    │   │
│      │  {                                              │   │
│      │      OnEntityAdded(id);     // 注册事件        │   │
│      │      OnEntityRemoved(id);  // 注销事件        │   │
│      │      OnEntityUpdated(id, u); // 更新事件        │   │
│      │  }                                              │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Strategy Pattern（策略模式）                          │
│      ─────────────────────────────────────                  │
│      索引是策略，Registry 是上下文                          │
│      不同索引实现不同的组织策略                              │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │  KeyedEntityIndex    ──► 一对一映射             │   │
│      │  MultiKeyEntityIndex ──► 一对多映射             │   │
│      │  CustomIndex         ──► 自定义策略            │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   4️⃣ Flyweight Pattern（享元模式）                        │
│      ─────────────────────────────────────                  │
│      用 int 类型作为 Entity ID 而不是完整对象                │
│      减少内存占用，便于在字典中快速查找                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 各模式解决的问题

| 模式 | 解决的问题 |
|------|------------|
| Mediator | 解耦 N 个实体和 M 个索引的复杂关系 |
| Observer | 索引自动响应实体变化，无需轮询 |
| Strategy | 支持多种索引策略（单键/多键/自定义） |
| Flyweight | 用 ID 代替完整对象，减少内存 |

## 六、使用指南

### 6.1 基础使用示例

```csharp
// 场景：管理战斗中的单位

public class BattleManager
{
    private readonly BattleEntityManager<int> _entityManager;

    // 索引定义
    private KeyedEntityIndex<Team, int> _byTeam;
    private KeyedEntityIndex<UnitType, int> _byType;
    private KeyedEntityIndex<int, int> _byOwner;  // 所有者
    private MultiKeyEntityIndex<int, int> _byBuff;  // Buff标签

    public BattleManager()
    {
        _entityManager = new BattleEntityManager<int>();

        // 创建索引
        _byTeam = _entityManager.CreateKeyedIndex<Team>(
            updateType: UpdateTypes.TeamChange);
        _byType = _entityManager.CreateKeyedIndex<UnitType>(
            updateType: UpdateTypes.TypeChange);
        _byOwner = _entityManager.CreateKeyedIndex<int>(
            updateType: UpdateTypes.OwnerChange);
        _byBuff = _entityManager.CreateMultiKeyIndex<int>(
            addKeyUpdateType: UpdateTypes.AddBuff,
            removeKeyUpdateType: UpdateTypes.RemoveBuff);
    }

    // 注册单位
    public void RegisterUnit(Unit unit)
    {
        _entityManager.Add(unit.Id);

        _byTeam.SetKey(unit.Id, unit.Team);
        _byType.SetKey(unit.Id, unit.Type);
        _byOwner.SetKey(unit.Id, unit.OwnerPlayerId);

        foreach (var buffId in unit.ActiveBuffs)
        {
            _byBuff.AddKey(unit.Id, buffId);
        }
    }

    // 查询：所有蓝队英雄
    public List<Unit> GetBlueTeamHeroes()
    {
        var blueUnits = _byTeam.Get(Team.Blue);
        var heroes = _byType.Get(UnitType.Hero);
        return blueUnits.Intersect(heroes).Select(FindUnit).ToList();
    }

    // 查询：某个玩家的所有单位
    public List<Unit> GetPlayerUnits(int playerId)
    {
        return _byOwner.Get(playerId).Select(FindUnit).ToList();
    }

    // 查询：所有有某个Buff的单位
    public List<Unit> GetUnitsWithBuff(int buffId)
    {
        return _byBuff.Get(buffId).Select(FindUnit).ToList();
    }

    // 单位转阵营
    public void ChangeTeam(int unitId, Team newTeam)
    {
        _entityManager.NotifyUpdated(unitId, new EntityUpdate
        {
            Type = UpdateTypes.TeamChange,
            Payload = new SetKeyUpdate<Team> { Key = newTeam }
        });
    }

    // 单位添加Buff
    public void AddBuff(int unitId, int buffId)
    {
        _entityManager.NotifyUpdated(unitId, new EntityUpdate
        {
            Type = UpdateTypes.AddBuff,
            Payload = new AddKeyUpdate<int> { Key = buffId }
        });
    }
}
```

### 6.3 自定义索引示例

```csharp
// 场景：创建一个"范围内单位"索引（按位置距离）

public class ProximityEntityIndex : IEntityIndex<int>
{
    private readonly Func<int, Vector3> _getPosition;
    private readonly Vector3 _center;
    private readonly float _range;

    private readonly Dictionary<int, float> _distanceCache =
        new Dictionary<int, float>();

    public ProximityEntityIndex(
        Func<int, Vector3> getPosition,
        Vector3 center,
        float range)
    {
        _getPosition = getPosition;
        _center = center;
        _range = range;
    }

    public void OnEntityAdded(int id)
    {
        UpdateDistance(id);
    }

    public void OnEntityRemoved(int id)
    {
        _distanceCache.Remove(id);
    }

    public void OnEntityUpdated(int id, EntityUpdate update)
    {
        UpdateDistance(id);
    }

    private void UpdateDistance(int id)
    {
        var pos = _getPosition(id);
        var dist = Vector3.Distance(pos, _center);
        _distanceCache[id] = dist;
    }

    public List<int> GetUnitsInRange()
    {
        var result = new List<int>();
        foreach (var kv in _distanceCache)
        {
            if (kv.Value <= _range)
            {
                result.Add(kv.Key);
            }
        }
        return result;
    }
}

// 使用
var proximityIndex = new ProximityEntityIndex(
    id => FindUnit(id).Position,
    center: transform.position,
    range: 10f
);
_entityManager.Registry.AddIndex(proximityIndex);
```
