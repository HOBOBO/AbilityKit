## 一、设计理念：两种触发器范式的对比

### 1.1 通用事件触发器 vs 强类型触发器

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        两种触发器范式对比                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  【通用事件触发器】（Ability.Triggering 包）                            │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                                                                  │  │
│  │  核心思想：事件 = 字符串ID + 泛型上下文                           │  │
│  │                                                                  │  │
│  │  EventId = "skill.cast.start"  // 字符串标识                    │  │
│  │  Context = TriggerContext              // 通用上下文              │  │
│  │                                                                  │  │
│  │  特点：                                                          │  │

│  │                                                                  │  │
```

### 2.3 触发器注册

```csharp
namespace AbilityKit.Ability.Triggering.Runtime
{
    /// <summary>
    /// 触发器注册表
    /// 支持从程序集自动注册条件和动作工厂
    /// </summary>
    public sealed class TriggerRegistry
    {
        private readonly Dictionary<string, IConditionFactory> _conditionFactories;
        private readonly Dictionary<string, IActionFactory> _actionFactories;

        /// <summary>
        /// 从程序集自动注册
        /// </summary>
        public void AutoRegisterFromAssemblies(params Assembly[] assemblies)
        {
            foreach (var type in assemblies.GetTypes())
            {
                // 查找 [TriggerActionType] 属性
                var actionAttr = type.GetCustomAttribute<TriggerActionTypeAttribute>();
                if (actionAttr != null && typeof(IActionFactory).IsAssignableFrom(type))
                {
                    var factory = CreateFactoryInstance<IActionFactory>(type);
                    RegisterAction(actionAttr.Type, factory);
                }

                // 查找 [TriggerConditionType] 属性
                var conditionAttr = type.GetCustomAttribute<TriggerConditionTypeAttribute>();
                if (conditionAttr != null && typeof(IConditionFactory).IsAssignableFrom(type))
                {
                    var factory = CreateFactoryInstance<IConditionFactory>(type);
                    RegisterCondition(conditionAttr.Type, factory);
                }
            }
        }

        /// <summary>
        /// 创建条件
        /// </summary>
        public ITriggerCondition CreateCondition(ConditionDef def)
        {
            if (!_conditionFactories.TryGetValue(def.Type, out var factory))
                throw new InvalidOperationException($"Condition type not registered: {def.Type}");
            return factory.Create(def);
        }

        /// <summary>
        /// 创建动作
        /// </summary>
        public ITriggerAction CreateAction(ActionDef def)
        {
            if (!_actionFactories.TryGetValue(def.Type, out var factory))
                throw new InvalidOperationException($"Action type not registered: {def.Type}");
            return factory.Create(def);
        }
    }
}
```

## 三、动作运行器设计

### 3.1 动作接口

```csharp

{
    /// <summary>
    /// 运行中的动作接口
    /// </summary>
    public interface IRunningAction : IDisposable
    {
        bool IsDone { get; }
        void Tick(float deltaTime);
        void Cancel();
    }
}
```

### 3.2 运行器实现

```csharp
namespace AbilityKit.Ability.Triggering.Runtime
{
    public sealed class TriggerActionRunner : ITriggerActionRunner
    {
        private struct Entry
        {
            public IRunningAction Action;
            public object Owner;
            public long OwnerKey;
        }

        private readonly List<Entry> _running = new List<Entry>(64);

        public void Add(IRunningAction action, long ownerKey)
        {
            _running.Add(new Entry { Action = action, OwnerKey = ownerKey });
        }

        public void Tick(float deltaTime)
        {
            for (int i = _running.Count - 1; i >= 0; i--)
            {
                var e = _running[i];
                var a = e.Action;

                if (a.IsDone)
                {
                    TryDispose(a);
                    _running.RemoveAt(i);
                    continue;
                }

                a.Tick(deltaTime);
            }
        }

        /// <summary>
        /// 按 OwnerKey 取消动作
        /// 用于 EffectSource 结束时取消所有关联动作
        /// </summary>
        public int CancelByOwnerKey(long ownerKey)
        {
            var count = 0;
            for (int i = _running.Count - 1; i >= 0; i--)
            {
                if (_running[i].OwnerKey != ownerKey) continue;

                _running[i].Action?.Cancel();
                TryDispose(_running[i].Action);
                _running.RemoveAt(i);
                count++;
            }
            return count;
        }
    }
}
```

### 4.2 条件实现示例

```csharp
namespace AbilityKit.Ability.Triggering.Runtime.Builtins
{
    /// <summary>
    /// Any 条件：任意子条件满足即通过
    /// </summary>
    [TriggerConditionType("any")]
    public sealed class AnyCondition : ITriggerCondition
    {
        private readonly List<ITriggerCondition> _children;

        public AnyCondition(IReadOnlyList<ITriggerCondition> children)
        {
            _children = new List<ITriggerCondition>(children);
        }

        public bool Evaluate(TriggerContext context)
        {
            foreach (var child in _children)
            {
                if (child.Evaluate(context)) return true;
            }
            return false;
        }

        public static AnyCondition FromDef(ConditionDef def, IConditionCompiler compiler)
        {
            var items = (List<ConditionDef>)def.Args["items"];
            var compiled = items.Select(c => compiler.Compile(c)).ToList();
            return new AnyCondition(compiled);
        }
    }

    /// <summary>
    /// 数值变量大于条件
    /// </summary>
    [TriggerConditionType("numeric.gt")]
    public sealed class NumericVarGreaterThanCondition : ITriggerCondition
    {
        private readonly NumericVarRef _var;
        private readonly double _threshold;

        public bool Evaluate(TriggerContext context)
        {
            if (context.TryGetNumericVar(_var, out var value))
            {
                return value > _threshold;
            }
            return false;
        }
    }
}
```

```csharp
namespace AbilityKit.Ability.Triggering.Variables.Numeric.Domains
{
    /// <summary>
    /// 局部变量域（绑定到 TriggerContext）
    /// </summary>
    public sealed class LocalNumericVarDomain : INumericVarDomain
    {
        private readonly Dictionary<string, double> _vars = new();

        public bool TryGet(TriggerContext context, string key, out double value)
        {
            return _vars.TryGetValue(key, out value);
        }

        public bool TrySet(TriggerContext context, string key, double value)
        {
            _vars[key] = value;
            return true;
        }
    }

    /// <summary>
    /// 全局变量域
    /// </summary>
    public sealed class GlobalNumericVarDomain : INumericVarDomain
    {
        private readonly Dictionary<string, double> _vars = new();
        private readonly object _lock = new();

        public bool TryGet(TriggerContext context, string key, out double value)
        {
            lock (_lock)
            {
                return _vars.TryGetValue(key, out value);
            }
        }

        public bool TrySet(TriggerContext context, string key, double value)
        {
            lock (_lock)
            {
                _vars[key] = value;
                return true;
            }
        }
    }
}
```

```csharp
namespace AbilityKit.Ability.Triggering.Variables.Numeric

    {
        public static bool TryGetNumericVar(
            this TriggerContext context,
            in NumericVarRef varRef,
            out double value)
        {
            var registry = context.Services.GetService<INumericVarDomainRegistry>();
            if (registry.TryGetDomain(varRef.DomainId, out var domain))
            {
                return domain.TryGet(context, varRef.Key, out value);
            }
            value = 0d;
            return false;

            this TriggerContext context,
            in NumericVarRef varRef,

        {
            var registry = context.Services.GetService<INumericVarDomainRegistry>();
            if (registry.TryGetDomain(varRef.DomainId, out var domain))
            {
                return domain.TrySet(context, varRef.Key, value);
            }
            return false;
        }
    }
```

```csharp
            if (ctx == null) return;

        }
```

## 七、两种实现对比

### 7.1 特性对比

| 特性 | 通用触发器 | 强类型触发器 |
|------|-----------|-------------|
| **类型安全** | ❌ 运行时检查 | ✅ 编译时检查 |
| **IDE 支持** | ❌ 无自动补全 | ✅ 完整自动补全 |
| **性能** | ⚠️ 需要反射/字典查找 | ✅ 直接调用 |
| **灵活性** | ✅ 支持运行时配置 | ❌ 硬编码 |
| **可读性** | ⚠️ 字符串ID需查文档 | ✅ 代码即文档 |
| **热更新** | ✅ 可从 JSON 驱动 | ❌ 需要代码修改 |
| **学习成本** | ⚠️ 较高 | ✅ 较低 |

### 8.2 架构原则

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        架构分层原则                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  【Rule 1：优先直接调用】                                              │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                                                                  │  │
│  │  不要为了"解耦"而引入事件系统                                   │  │
│  │  只有当直接调用导致问题时才使用事件                            │  │
│  │                                                                  │  │
│  │  ❌ 过度设计：                                                  │  │
│  │     OnDamage() → Publish("unit.damaged", ctx) → OnUnitDamaged()│  │
│  │                                                                  │  │
│  │  ✅ 简单直接：                                                  │  │
│  │     OnDamage(target, damage) → target.Health -= damage;        │  │
│  │                                                                  │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  【Rule 2：强类型优先】                                                │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                                                                  │  │
│  │  优先使用强类型触发器                                          │  │
│  │  只有在需要配置驱动时才使用通用触发器                         │  │
│  │                                                                  │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  【Rule 3：按需引入】                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                                                                  │  │
│  │  功能复杂度增加路径：                                          │  │
│  │  直接调用 → 强类型事件 → 通用触发器                        │  │
│  │                                                                  │  │
│  │  只有当低层方案无法满足需求时才升级到更高层                   │  │
│  │                                                                  │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 九、设计总结

### 9.1 核心优势

| 触发器类型 | 优势 |
|------------|------|
| **通用触发器** | 完全解耦、配置驱动、热更新支持 |
| **强类型触发器** | 类型安全、性能好、IDE 支持 |
| **直接调用** | 零开销、最简单 |

### 9.2 选择决策树

```
开始
  │
  ├─ 需要运行时配置吗？
  │    │
  │    ├─ 是 ──▶ 通用触发器
  │    │
  │    └─ 否 ──┬─ 订阅方和发布方是否必须解耦？
  │              │
  │              ├─ 是 ──▶ 强类型触发器
  │              │
  │              └─ 否 ──▶ 直接方法调用
  │
  └─ 性能是否极度敏感？
       │
       ├─ 是 ──▶ 直接方法调用
       │
       └─ 否 ──▶ 继续上面的决策
```
