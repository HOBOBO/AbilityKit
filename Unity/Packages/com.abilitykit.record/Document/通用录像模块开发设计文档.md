# Ability-Kit 通用录像模块开发设计文档

> **阅读对象**：首次接触 Ability-Kit 录像模块的开发者
>
> **文档目标**：让你理解录像模块"是什么"、"解决了什么问题"、"为什么这样设计"、"怎么使用"

## 一、设计理念：为什么要做录像模块？

### 1.1 游戏开发中的痛点

```
❌ 传统做法的问题：

1. 帧同步游戏难以调试
   ┌─────────────────────────────────────────────────────────┐
   │ // 玩家A和玩家B看到的画面不一致                         │
   │ // 怎么知道是哪一帧出了问题？                          │
   │ // 服务器日志？客户端日志？两边的状态怎么对比？         │
   └─────────────────────────────────────────────────────────┘

2. Bug难以复现
   ┌─────────────────────────────────────────────────────────┐
   │ // "我这里闪退了，你那里呢？"                         │
   │ // "我这里没事啊..."                                  │
   │ // "那我再打一局试试..."                              │
   │ // 同样的操作，可能永远无法复现                        │
   └─────────────────────────────────────────────────────────┘

3. 状态同步难以验证
   ┌─────────────────────────────────────────────────────────┐
   │ // 客户端预测的结果和服务器一致吗？                    │
   │ // 怎么验证"确定性"是否真的被保证了？                 │
   └─────────────────────────────────────────────────────────┘

4. 战斗回放难以实现
   ┌─────────────────────────────────────────────────────────┐
   │ // 想做一个"战斗回放"功能                             │
   │ // 需要记录什么？输入？状态？事件？                    │
   │ // 怎么存储？用什么格式？                             │
   └─────────────────────────────────────────────────────────┘
```

### 1.2 录像模块的解决方案

```
✅ 录像模块的设计思路：

┌─────────────────────────────────────────────────────────────┐
│                       核心思想                              │
│                                                             │
│   把"游戏过程"录制下来，像录像带一样保存                    │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   录制模式：                                          │  │
│   │   ┌───────────────────────────────────────────┐    │  │
│   │   │ Frame 1: 玩家A输入"移动"                  │    │  │
│   │   │ Frame 2: 玩家B输入"攻击"                  │    │  │
│   │   │ Frame 3: 状态哈希 = 0xABCD1234            │    │  │
│   │   │ Frame 4: 玩家A输入"放技能"                │    │  │
│   │   │ ...                                         │    │  │
│   │   └───────────────────────────────────────────┘    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   回放模式：                                          │  │
│   │   从头播放录制内容，验证每帧状态                       │  │
│   │   - 可以定位到具体哪一帧出错                          │  │
│   │   - 可以对比客户端和服务器的哈希                      │  │
│   │   - 可以复现Bug                                      │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                    录像模块的三大承诺                        │
│                                                             │
│   1️⃣ 确定性友好                                          │
│      录制的是输入+状态哈希，不是最终状态                    │
│      保证"同样的输入，一定产生同样的结果"                   │
│                                                             │
│   2️⃣ 模块化设计                                          │
│      输入、快照、日志是独立的 Track（轨道）                │
│      按需录制，按需回放                                   │
│                                                             │
│   3️⃣ 可扩展架构                                          │
│      自定义事件类型、自定义编解码器                        │
│      不绑定具体游戏类型                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解录像模块

### 2.1 什么是"录制（Record）"？

**通俗解释**：录制就是把游戏过程中的重要事件按时间顺序记录下来。

```
┌─────────────────────────────────────────────────────────────┐
│                    录制的组成                                │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                 RecordSession (录像会话)             │  │
│   │                                                       │  │
│   │   ┌─────────────────────────────────────────────┐   │  │
│   │   │              RecordContainer (录像带)         │   │  │
│   │   │                                               │   │  │
│   │   │   Track: inputs                               │   │  │
│   │   │   ├─► Frame 1: Input(移动)                   │   │  │
│   │   │   ├─► Frame 2: Input(攻击)                   │   │  │
│   │   │   └─► Frame 3: Input(技能)                   │   │  │
│   │   │                                               │   │  │
│   │   │   Track: state_hash                          │   │  │
│   │   │   ├─► Frame 1: Hash=0x1234                  │   │  │
│   │   │   ├─► Frame 3: Hash=0x5678                  │   │  │
│   │   │   └─► Frame 5: Hash=0xABCD                  │   │  │
│   │   │                                               │   │  │
│   │   └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 什么是"轨道（Track）"？

**通俗解释**：Track 就像录像带的不同声道，每条轨道记录不同类型的信息。

```
┌─────────────────────────────────────────────────────────────┐
│                    多轨道录制                                │
│                                                             │
│   同一时间点，可以有多个事件同时记录                        │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   Frame 1 ──┬──► Track: inputs ───► [移动命令]       │  │
│   │             │                                        │  │
│   │             └──► Track: snapshots ─► [完整状态]       │  │
│   │                                                       │  │
│   │   Frame 2 ──┬──► Track: inputs ───► [攻击命令]       │  │
│   │             │                                        │  │
│   │             └──► Track: state_hash ─► [状态哈希]    │  │
│   │                                                       │  │
│   │   Frame 3 ──┬──► Track: inputs ───► [技能命令]       │  │
│   │             │                                        │  │
│   │             └──► Track: battle_log ─► [战斗事件]     │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   好处：                                                    │
│   - 不同信息可以独立处理                                    │
│   - 回放时可以选择性读取                                    │
│   - 便于扩展新类型的录制                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **RecordSession** | 一次录制的会话，管理所有数据 | 录像机的控制面板 |
| **RecordContainer** | 录像数据本身，可序列化 | 录像带 |
| **RecordTrack** | 一条录制轨道，某个领域的时间序列 | 录像带的一个声道 |
| **RecordEvent** | 轨道上的一个事件点 | 录像中的一个画面 |
| **FrameIndex** | 帧编号，表示事件发生的时间点 | 录像的时间戳 |
| **EventType** | 事件类型，区分不同种类的事件 | 画面内容标签 |
| **Codec** | 编解码器，把事件转换为二进制 | 视频压缩算法 |

## 四、核心组件详解

### 4.1 RecordSession - 录像会话

**通俗解释**：Session 就像录像机的控制面板，提供了所有操作入口。

```
┌─────────────────────────────────────────────────────────────┐
│                    RecordSession 接口                        │
│                                                             │
│   public interface IRecordSession : IDisposable              │
│   {                                                         │
│       // 配置                                                │
│       RecordProfile Profile { get; }                        │
│       RecordContainer Container { get; }                    │
│                                                             │
│       // 写入                                                │
│       bool TryGetWriter(TrackId, out IEventTrackWriter)    │
│                                                             │
│       // 读取                                                │
│       bool TryGetReader(TrackId, out IEventTrackReader)    │
│                                                             │
│       // 持久化                                              │
│       byte[] Serialize();                                   │
│       bool TryLoad(byte[] data);                           │
│   }                                                         │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   典型使用流程：                                       │  │
│   │                                                       │  │
│   │   创建会话 ──► 获取Writer ──► 录制 ──► 序列化保存     │  │
│   │   加载文件  ──► 获取Reader ──► 回放                   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 RecordContainer - 录像数据容器

**通俗解释**：Container 是录像带本身，包含所有录制的数据。

```
┌─────────────────────────────────────────────────────────────┐
│                    RecordContainer 结构                      │
│                                                             │
│   public sealed class RecordContainer                       │
│   {                                                         │
│       // 元数据（游戏版本、时间、玩家信息等）                 │
│       public Dictionary<string, object> Meta { get; set; } │
│                                                             │
│       // 所有轨道                                             │
│       public Dictionary<RecordTrackId, RecordTrack>         │
│           Tracks { get; set; }                             │
│   }                                                         │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   Meta 示例：                                         │  │
│   │   {                                                   │  │
│   │       "game_version": "1.0.0",                       │  │
│   │       "record_time": "2026-03-19T10:00:00Z",         │  │
│   │       "player_count": 10,                            │  │
│   │       "map_id": 1001                                 │  │
│   │   }                                                   │  │
│   │                                                       │  │
│   │   Tracks 示例：                                       │  │
│   │   {                                                   │  │
│   │       "inputs": { ... },                             │  │
│   │       "snapshots": { ... },                         │  │
│   │       "battle_log": { ... }                          │  │
│   │   }                                                   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 RecordTrack - 录制轨道

**通俗解释**：Track 是录像带的一个声道，存储一类事件的完整时间序列。

## 六、使用指南

### 6.1 基础录制示例

```csharp
// 场景：录制一局游戏的输入

public class GameRecorder
{
    private IRecordSession _session;
    private IEventTrackWriter _inputWriter;

    public void StartRecording()
    {
        // 1. 创建录制会话
        var profile = new RecordProfile
        {
            EnableInputs = true,
            EnableStateHash = true
        };
        _session = RecordSessionFactory.CreateDefault(profile);

        // 2. 获取输入轨道的写入器
        var trackId = RecordTrackId.FromName("inputs");
        _session.TryGetWriter(trackId, out _inputWriter);

        // 3. 添加元数据
        _session.Container.Meta["game_version"] = "1.0.0";
        _session.Container.Meta["player_count"] = PlayerCount;
    }

    public void RecordFrame(int frameNumber, PlayerInput[] inputs)
    {
        var frame = new FrameIndex(frameNumber);

        foreach (var input in inputs)
        {
            var payload = InputCommandCodec.Encode(input);
            _inputWriter.Append(frame, RecordEventTypes.InputCommand, payload);
        }
    }

    public void StopAndSave(string path)
    {
        var bytes = _session.Serialize();
        RecordFileStore.Save(path, bytes);
        _session.Dispose();
    }
}
```

### 6.2 基础回放示例

```csharp
// 场景：回放录制的游戏

public class GameReplayer
{
    private IRecordSession _session;
    private IReplayController _controller;
    private IEventTrackReader _inputReader;

    public void LoadRecording(string path)
    {
        // 1. 加载录制文件
        var bytes = RecordFileStore.Load(path);
        _session = RecordSessionFactory.LoadFromBytes(bytes);

        // 2. 获取输入轨道的读取器
        var trackId = RecordTrackId.FromName("inputs");
        _session.TryGetReader(trackId, out _inputReader);

        // 3. 创建回放控制器
        var clock = new FixedStepReplayClock(fixedDelta: 1f / 30f);
        var handler = new InputReplayHandler();
        _controller = new BasicReplayController(clock, _inputReader, handler);
    }

    public void StartReplay()
    {
        _controller.Play();
    }

    public void Update()
    {
        // 每帧调用 Tick
        _controller.Tick(Time.deltaTime);
    }
}

// 事件处理器
public class InputReplayHandler : IReplayEventHandler
{
    public void Handle(in RecordEvent e)
    {
        var input = InputCommandCodec.Decode(e.Payload);
        Debug.Log($"Frame {e.Frame.Value}: Player {input.PlayerId} - OpCode {input.OpCode}");
    }
}
```

```csharp
// 场景：录制战斗日志

// 1. 定义事件类型常量
public static class BattleLogEventTypes
{
    public static readonly RecordEventType SkillCast =
        RecordEventType.FromName("battle.skill.cast");

    public static readonly RecordEventType DamageDealt =
        RecordEventType.FromName("battle.damage.dealt");

    public static readonly RecordEventType UnitDied =
        RecordEventType.FromName("battle.unit.died");
}

// 2. 定义 Codec
public static class SkillCastEventCodec
{
    readonly struct Payload
    {
        [BinaryMember(0)] public readonly int CasterId;
        [BinaryMember(1)] public readonly int SkillId;
        [BinaryMember(2)] public readonly int TargetId;
    }

    public static byte[] Encode(int casterId, int skillId, int targetId)
    {
        var p = new Payload(casterId, skillId, targetId);
        return BinaryObjectCodec.Encode(p);
    }

    public static (int, int, int) Decode(byte[] payload)
    {
        var p = BinaryObjectCodec.Decode<Payload>(payload);
        return (p.CasterId, p.SkillId, p.TargetId);
    }
}

// 3. 使用
public void RecordSkillCast(int frame, int casterId, int skillId, int targetId)
{
    var payload = SkillCastEventCodec.Encode(casterId, skillId, targetId);
    _writer.Append(new FrameIndex(frame), BattleLogEventTypes.SkillCast, payload);
}
```

```
│      │  │ │ │ EventTrack                      │ │  │   │   │
│      │  │ │ └─────────────────────────────────┘ │  │   │   │
│      │  │ └─────────────────────────────────────┘  │   │   │
```
