## 一、设计理念：为什么要做移动系统模块？

### 1.1 传统做法的问题

```
❌ 传统做法的移动系统问题：

1. 移动逻辑堆砌在一起
   ┌─────────────────────────────────────────────────────────┐
   │ // CharacterController 的问题：                          │
   │ void Update() {                                        │
   │     if (isStunned) return;                            │
   │     if (isKnockback) { DoKnockback(); return; }      │
   │     if (isDashing) { DoDash(); return; }              │
   │     if (isPathfinding) { DoPathfinding(); return; }   │
   │     DoNormalMove();                                   │
   │ }                                                     │
   │                                                          │
   │ // 每个新移动效果都要加 if                             │
   │ // 条件多了就变成面条代码                              │
   │ // 优先级混乱，难以维护                                │
   └─────────────────────────────────────────────────────────┘

2. 多个移动效果无法叠加
   ┌─────────────────────────────────────────────────────────┐
   │ // 问题是：只能有一个移动生效                         │
   │ // 被击退时不能走位                                    │
   │ // 被控时不能施法                                      │
   │                                                          │
   │ if (isKnockback) {                                    │
   │     // 强制位移优先                                    │
   │ } else if (isDashing) {                              │
   │     // 冲刺优先                                        │
   │ } else {                                             │
   │     // 普通移动                                        │
   │ }                                                    │
   │                                                          │
   │ // 实际上应该有更灵活的组合方式                       │
   └─────────────────────────────────────────────────────────┘

3. 移动与碰撞耦合
   ┌─────────────────────────────────────────────────────────┐
   │ // Unity 的 CharacterController                      │
   │ // 移动、碰撞、地面检测混在一起                        │
   │ // 想换碰撞系统？重写                                  │
   │ // 想做服务端验证？困难                                │
   └─────────────────────────────────────────────────────────┘

4. 不确定性
   ┌─────────────────────────────────────────────────────────┐
   │ // 直接调用 Transform.Translate                       │
   │ // 帧率不同结果不同                                   │
   │ // 无法用于帧同步回放                                  │
   └─────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解移动系统

### 2.1 什么是"移动来源（MotionSource）"？

**通俗解释**：MotionSource 是"移动的来源"，每个来源产生自己的位移需求。

```
┌─────────────────────────────────────────────────────────────┐
│                    MotionSource = 移动的来源                 │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  每个移动效果都是一个来源                             │  │
│   │                                                       │  │
│   │  ┌───────────────┐ ┌───────────────┐               │  │
│   │  │ 输入移动      │ │ 冲刺位移      │               │  │
│   │  │ Locomotion    │ │ Trajectory    │               │  │
│   │  │ 玩家WASD输入   │ │ 技能轨迹      │               │  │
│   │  └───────────────┘ └───────────────┘               │  │
│   │  ┌───────────────┐ ┌───────────────┐               │  │
│   │  │ 击退位移      │ │ 路径跟随      │               │  │
│   │  │ FixedDelta    │ │ PathFollower  │               │  │
│   │  │ 固定方向速度   │ │ 自动寻路      │               │  │
│   │  └───────────────┘ └───────────────┘               │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  MotionSource 的属性                                │  │
│   │                                                       │  │
│   │  GroupId   属于哪个组（Locomotion/Ability/Control/Path）│  │
│   │  Priority  优先级（组内Exclusive时使用）              │  │
│   │  Stacking  叠加方式（Additive/Exclusive/Override）    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **MotionSource** | 移动的来源 | 动力来源（引擎、肌肉、风） |
| **MotionPipeline** | 移动协调器 | 交通指挥中心 |
| **MotionSolver** | 碰撞/约束求解器 | 交通规则执行者 |
| **MotionState** | 移动状态 | 车辆当前状态（位置、速度、方向） |
| **MotionOutput** | 移动结果 | 实际行驶距离 |
| **MotionGroups** | 移动分组 | 车辆类型（普通车、救护车、警车） |
| **MotionStacking** | 叠加策略 | 同行规则（可以并行/需要让行） |

## 四、核心组件详解

### 4.1 MotionPipeline - 移动协调器

**通俗解释**：Pipeline 是"交通指挥中心"，协调所有移动来源。

```
┌─────────────────────────────────────────────────────────────┐
│                    MotionPipeline 结构                         │
│                                                             │

│   │   public sealed class MotionPipeline                  │  │
│   │   {                                                  │  │
│   │       public IMotionSolver Solver;                   │  │
│   │       public MotionPipelinePolicy Policy;             │  │
│   │       public MotionState State;                       │  │
│   │                                                       │  │
│   │       public void AddSource(IMotionSource source);   │  │
│   │       public void RemoveSource(IMotionSource source); │  │

│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
```

### 4.2 IMotionSource - 移动来源接口

**通俗解释**：Source 是"动力"，每个来源产生自己的位移。

### 4.4 MotionStacking - 叠加策略

**通俗解释**：Stacking 是"叠加规则"，决定同组来源如何相互作用。

## 五、设计模式总结

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Strategy Pattern（策略模式）                          │
│      ─────────────────────────────────────                  │
│      IMotionSource、IMotionSolver 都是策略接口              │
│      可替换实现                                             │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   IMotionSource:                                  │   │
│      │   ├─► LocomotionMotionSource   (输入移动)       │   │
│      │   ├─► TrajectoryMotionSource   (轨迹移动)       │   │
│      │   ├─► FixedDeltaMotionSource   (固定位移)       │   │
│      │   └─► PathFollowerMotionSource (路径跟随)       │   │
│      │                                                   │   │
│      │   IMotionSolver:                                  │   │
│      │   ├─► NoMotionSolver           (无碰撞)         │   │
│      │   └─► ConfigurableMotionSolver (可配置碰撞)      │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Decorator Pattern（装饰器模式）                       │
│      ─────────────────────────────────────                  │
│      ScaledMotionSource 包装其他 Source                      │
│      添加缩放功能，不改变原有行为                            │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   ScaledMotionSource                            │   │
│      │   ├─► inner: IMotionSource                     │   │
│      │   └─► scale: 0.7                               │   │
│      │                                                   │   │
│      │   outDesiredDelta = inner.Tick() * 0.7          │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Pipeline Pattern（管线模式）                          │
│      ─────────────────────────────────────                  │
│      MotionPipeline 协调多个阶段的处理                      │
│      收集 → 过滤 → 求解 → 输出                              │
│                                                             │
│   4️⃣ Composition Pattern（组合模式）                        │
│      ─────────────────────────────────────                  │
│      多个 MotionSource 组合使用                            │
│      Additive 模式累加效果                                 │
│                                                             │
│   5️⃣ Observer Pattern（观察者模式）                        │
│      ─────────────────────────────────────                  │
│      IMotionEventSink 接收完成事件                         │
│      IMotionFinishEventSource 产生事件                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 各模式解决的问题

| 模式 | 解决的问题 |
|------|------------|
| Strategy | 灵活替换移动和碰撞实现 |
| Decorator | 可选地包装和增强来源 |
| Pipeline | 协调多个处理阶段 |
| Composition | 多来源组合 |
| Observer | 解耦事件通知 |

## 六、使用指南

### 6.1 基础使用示例

```csharp
// 场景：角色移动系统

public class CharacterMotion
{
    private readonly MotionPipeline _pipeline;
    private readonly LocomotionMotionSource _locomotion;

    public CharacterMotion()
    {
        _pipeline = new MotionPipeline
        {
            Solver = new ConfigurableMotionSolver(world),
            Policy = MotionPipelinePolicy.CreateDefault()
        };

        _locomotion = new LocomotionMotionSource(
            speed: 5f,
            space: MotionInputSpace.Local  // 相对朝向移动
        );

        _pipeline.AddSource(_locomotion);
    }

    // 每帧调用
    public void Update(Vec3 position, Vec3 forward, float dt)
    {
        var state = new MotionState
        {
            Position = position,
            Forward = forward
        };

        _locomotion.SetInput(_inputX, _inputZ);

        var output = new MotionOutput();
        _pipeline.Tick(entityId, ref state, dt, ref output);

        // 应用结果
        _transform.Position = state.Position;
        _transform.Forward = state.Forward;
    }
}
```

```csharp
public void CastDash(Vec3 start, Vec3 direction, float distance, float duration)
{

    var trajectory = new LinearTrajectory3D(
        start: start,
        end: start + direction * distance,
        duration: duration
    );

    // 创建轨迹移动源
    var dashSource = new TrajectoryMotionSource(
        trajectory: trajectory,
        priority: 50,  // 高于 Locomotion (0)
        groupId: MotionGroups.Ability,
        stacking: MotionStacking.ExclusiveHighestPriority  // 独占
    );

    // 监听完成事件
    dashSource.OnArrive += () => Debug.Log("冲刺完成");

    // 添加到管道
    _pipeline.AddSource(dashSource);
}
```

```csharp
    {

    // Policy 会自动抑制 Locomotion、Ability、Path 组
    // 玩家在击退期间无法控制角色
}
```

### 6.5 自定义碰撞求解器示例

```csharp
// 场景：自定义碰撞逻辑（如穿透某些墙）

public class CustomMotionSolver : IMotionSolver
{
    private readonly IMotionCollisionWorld _world;
    private readonly uint _obstacleMask;
    private readonly uint _passThroughMask;

    public MotionSolveResult Solve(int id, in MotionState state,
        in MotionOutput input, float dt)
    {
        // 尝试扫掠检测
        if (_world.Sweep(id, state.Position, input.DesiredDelta,
            radius: 0.5f,
            obstacleMask: _obstacleMask,
            ignoreMask: _passThroughMask,
            out var hit, out var applied))
        {
            // 墙壁碰撞，停止或滑行
            if (hit.Material.IsSlippery)
            {
                // 滑行：沿墙面滑动
                return SlideAlongSurface(hit, input.DesiredDelta);
            }
            else
            {
                // 停止：完全停在墙前
                return MotionSolveResult.StopAtHit(applied, hit);
            }
        }

        // 无碰撞
        return MotionSolveResult.NoHit(input.DesiredDelta);
    }
}
```
