## 一、设计理念：为什么要做技能库模块？

### 1.1 传统做法的问题

```
❌ 传统做法的问题：

1. 技能查询效率低下
   ┌─────────────────────────────────────────────────────────┐
   │ // 需要找到"所有火系技能"                               │
   │ List<Skill> GetFireSkills() {                           │
   │     var result = new List<Skill>();                    │
   │     foreach (var skill in allSkills) {                  │
   │         if (skill.School == SkillSchool.Fire) {         │
   │             result.Add(skill);                          │
   │         }                                               │
   │     }                                                   │
   │     return result;                                      │
   │ }                                                       │
   │                                                          │
   │ // 1000个技能 × 10种查询 = 每次都是O(N)遍历           │
   │ // 帧同步游戏，每帧都要查，CPU爆炸                      │
   └─────────────────────────────────────────────────────────┘

2. 技能分类散落各处
   ┌─────────────────────────────────────────────────────────┐
   │ // SkillManager: 管理技能数据                           │
   │ // SkillBook: 按学院分类                               │
   │ // SkillTree: 按系谱分类                               │
   │ // SkillTagSystem: 按标签分类                          │
   │ // 分类逻辑分散，重复代码一堆                          │
   └─────────────────────────────────────────────────────────┘

3. 技能更新后分类不同步
   ┌─────────────────────────────────────────────────────────┐
   │ // 技能升级后，从"初级火球"变成"高级火球"            │
   │ skill.Level = 5;  // ← 火系学院列表不会自动更新！    │
   │ // 继续在初级列表里找到它                             │
   └─────────────────────────────────────────────────────────┘

4. 多维度查询困难
   ┌─────────────────────────────────────────────────────────┐
   │ // "找火系+范围攻击+带控制效果的技能"                 │
   │ // 要遍历所有技能，一个个条件筛选？                    │
   │ // 太慢了！                                            │
   └─────────────────────────────────────────────────────────┘
```

### 1.2 技能库模块的解决方案

```
✅ 技能库的设计思路：

┌─────────────────────────────────────────────────────────────┐
│                       核心思想                              │
│                                                             │
│   用"索引表"预先组织技能，查询时直接查表                    │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   注册技能时：                                        │  │
│   │   ┌───────────────────────────────────────────┐    │  │
│   │   │  skill.School = Fire;                    │    │  │
│   │   │  skill.Tags = [Damage, AoE, Stun];       │    │  │
│   │   │  lib.Add(skillId, skill);                 │    │  │
│   │   │                                           │    │  │
│   │   │  自动同步到索引表：                       │    │  │
│   │   │  ├─► 按学院: Fire → [火球, 火墙, ...]   │    │  │
│   │   │  ├─► 按标签: Damage → [...], Stun → [...]│    │  │
│   │   │  └─► 按等级: Level5 → [火球升级版]       │    │  │
│   │   └───────────────────────────────────────────┘    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   查询时：                                            │  │
│   │   ┌───────────────────────────────────────────┐    │  │
│   │   │  var fireSkills = bySchool.Get(Fire);   │    │  │
│   │   │  var stunSkills = byTag.Get(Stun);      │    │  │
│   │   │                                           │    │  │
│   │   │  // O(1) 字典查找，不是遍历               │    │  │
│   │   └───────────────────────────────────────────┘    │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                    技能库模块的三大承诺                        │
│                                                             │
│   1️⃣ 查询高效                                            │
│      用索引表替代遍历，查询是O(1)字典查找                   │
│                                                             │
│   2️⃣ 自动同步                                            │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心概念：从零理解技能库

### 2.1 什么是"技能库（SkillLibrary）"？

**通俗解释**：SkillLibrary 是"技能图书馆"，管理所有技能数据，并提供多个"分类目录"供查询。

```
┌─────────────────────────────────────────────────────────────┐
│                    技能库 = 图书馆                          │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  SkillLibrary<SkillId, SkillData>                    │  │
│   │                                                       │  │
│   │  核心存储：                                          │  │
│   │  _skills: Dictionary<SkillId, SkillData>            │  │
│   │                                                       │  │
│   │  索引目录：                                          │  │
│   │  _indices: List<ISkillIndex>                         │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   索引目录示例：                                      │  │
│   │   ┌─────────────────────────────────────────────┐   │  │
│   │   │                                               │   │  │
│   │   │  按学院分类 (KeyedIndex)：                    │   │  │
│   │   │  ├─► 火系: [火球, 火墙, 火焰风暴]         │   │  │
│   │   │  ├─► 冰系: [冰箭, 冰墙, 暴风雪]           │   │  │
│   │   │  └─► 雷系: [闪电, 雷击, 连锁闪电]         │   │  │
│   │   │                                               │   │  │
│   │   │  按标签分类 (MultiKeyIndex)：                │   │  │
│   │   │  ├─► 伤害: [火球, 冰箭, 闪电, ...]       │   │  │
│   │   │  ├─► 控制: [冰墙, 雷击, 束缚]            │   │  │
│   │   │  └─► 治疗: [治疗术, 愈合, 复生]          │   │  │
│   │   │                                               │   │  │
│   │   └─────────────────────────────────────────────┘   │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键名词解释

| 名词 | 通俗解释 | 类比 |
|------|----------|------|
| **SkillLibrary** | 技能库，管理技能数据和索引 | 图书馆 |
| **SkillIndex** | 索引，观察者接口 | 目录观察员 |
| **KeyedSkillIndex** | 单键索引，一个技能一个分类键 | "按学院分类"（每个技能只属一个学院） |
| **MultiKeySkillIndex** | 多键索引，一个技能可以有多个标签 | "按标签分类"（一个技能可以有多个标签） |
| **SkillUpdate** | 技能更新事件 | 属性变更通知单 |

## 四、核心组件详解

### 4.1 SkillLibrary - 技能库容器

**通俗解释**：SkillLibrary 是"图书馆管理员"，管理所有技能和索引。

```
┌─────────────────────────────────────────────────────────────┐
│                    SkillLibrary 结构                          │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   public class SkillLibrary<TKey, TData>             │  │
│   │   {                                                  │  │
│   │       private Dictionary<TKey, TData> _skills;       │  │
│   │       private List<ISkillIndex> _indices;            │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   核心方法：                                          │  │
│   │                                                       │  │
│   │   Add(key, data)                                      │  │
│   │   ─────────────────                                   │  │
│   │   1. _skills[key] = data;                            │  │
│   │   2. 遍历 _indices，index.OnAdded(key, data)         │  │
│   │                                                       │  │
│   │   Remove(key)                                         │  │
│   │   ──────────────                                      │  │
│   │   1. var data = _skills[key];                        │  │
│   │   2. _skills.Remove(key);                            │  │
│   │   3. 遍历 _indices，index.OnRemoved(key, data)       │  │
│   │                                                       │  │
│   │   Update(key, newData, update)                        │  │
│   │   ───────────────────────────────                     │  │
│   │   1. var oldData = _skills[key];                     │  │
│   │   2. _skills[key] = newData;                        │  │
│   │   3. 遍历 _indices，index.OnUpdated(key, oldData,   │  │
│   │                                  newData, update)   │  │
│   │                                                       │  │
```

### 4.2 ISkillIndex - 观察者接口

**通俗解释**：Index 是"目录观察员"，监听技能库的变化并更新自己的目录。

```
│   │                                                       │  │
```

### 4.4 DerivedKeyedSkillIndex - 单键索引

**通俗解释**：DerivedKeyedIndex 是"单标签分类目录"，每个技能只有一个分类标签。

```
┌─────────────────────────────────────────────────────────────┐
│                    DerivedKeyedSkillIndex 结构               │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   private readonly Func<TData, TIndexKey> _selector;│  │
│   │   private readonly Dictionary<TIndexKey,             │  │
│   │       HashSet<TKey>> _map;                          │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   OnAdded(key, data)                                 │  │
│   │   ─────────────────                                   │  │
│   │   var indexKey = _selector(data);  // 提取索引键   │  │
│   │   _map[indexKey].Add(key);              // 加入映射 │  │
│   │                                                       │  │
│   │   OnUpdated(key, oldData, newData, update)            │  │
│   │   ─────────────────────────────────────               │  │
│   │   var oldKey = _selector(oldData);                   │  │
│   │   var newKey = _selector(newData);                    │  │
│   │                                                       │  │
│   │   if (oldKey != newKey) {                            │  │
│   │       _map[oldKey].Remove(key);  // 从旧键移除      │  │
│   │       _map[newKey].Add(key);     // 加入新键        │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   使用场景：                                                │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  - 学院索引：每个技能只属一个学院                    │  │
│   │  - 元素索引：每个技能只属一种元素                    │  │
│   │  - 职业索引：每个技能只属一个职业                    │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.5 DerivedMultiKeySkillIndex - 多键索引

**通俗解释**：DerivedMultiKeyIndex 是"多标签分类目录"，一个技能可以有多个标签。

```
┌─────────────────────────────────────────────────────────────┐
│                    DerivedMultiKeySkillIndex 结构            │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   private readonly Func<TData,                       │  │
│   │       IEnumerable<TIndexKey>> _selector;              │  │
│   │   private readonly Dictionary<TIndexKey,             │  │
│   │       HashSet<TKey>> _map;                           │  │
│   │   private readonly IEqualityComparer<TIndexKey>      │  │
│   │       _keyComparer;                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                       │  │
│   │   OnUpdated(key, oldData, newData, update)            │  │
│   │   ─────────────────────────────────────               │  │
│   │   var oldKeys = ExtractKeys(oldData);  // HashSet   │  │
│   │   var newKeys = ExtractKeys(newData);  // HashSet   │  │
│   │                                                       │  │
│   │   // 计算差集                                         │  │
│   │   foreach (var k in oldKeys - newKeys) {             │  │
│   │       _map[k].Remove(key);  // 移除失效标签          │  │
│   │   }                                                  │  │
│   │   foreach (var k in newKeys - oldKeys) {             │  │
│   │       _map[k].Add(key);    // 添加新标签            │  │
│   │   }                                                  │  │
│   │                                                       │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
│   使用场景：                                                │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  - 标签索引：一个技能可以有多个标签                  │  │
│   │    例：[伤害, 火系, 范围攻击, 控制]                 │  │
│   │  - 效果索引：一个技能可以有多个效果标签              │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 五、设计模式总结

### 5.1 模块使用的设计模式

```
┌─────────────────────────────────────────────────────────────┐
│                    设计模式应用                              │
│                                                             │
│   1️⃣ Observer Pattern（观察者模式）                        │
│      ─────────────────────────────────────                  │
│      索引观察 SkillLibrary 的技能生命周期事件                │
│      收到通知后更新自己的内部状态                            │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │   SkillLibrary                                   │   │
│      │      │                                            │   │
│      │      │ OnAdded/OnRemoved/OnUpdated               │   │
│      │      │                                            │   │
│      │      ▼                                            │   │
│      │   ┌───────────────────────────────────────────┐  │   │
│      │   │  ISkillIndex[]                            │  │   │
│      │   │  ├─► bySchool (观察者)                    │  │   │
│      │   │  ├─► byTag (观察者)                      │  │   │
│      │   │  └─► byLevel (观察者)                    │  │   │
│      │   └───────────────────────────────────────────┘  │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   2️⃣ Strategy Pattern（策略模式）                          │
│      ─────────────────────────────────────                  │
│      索引是策略，SkillLibrary 是上下文                      │
│      不同索引实现不同的组织策略                              │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │  DerivedKeyedSkillIndex    ──► 一对一映射     │   │
│      │  DerivedMultiKeySkillIndex ──► 一对多映射     │   │
│      │  CustomSkillIndex         ──► 自定义策略    │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   3️⃣ Factory Method Pattern（工厂方法模式）                 │
│      ────────────────────────────────────────────────────  │
│      SkillLibrary 提供工厂方法创建索引                       │
│      封装索引的创建细节                                      │
│                                                             │
│      ┌─────────────────────────────────────────────────┐   │
│      │                                                   │   │
│      │  lib.CreateDerivedKeyedIndex<T>(selector)      │   │
│      │  lib.CreateDerivedMultiKeyIndex<T>(selector)   │   │
│      │                                                   │   │
│      └─────────────────────────────────────────────────┘   │
│                                                             │
│   4️⃣ Flyweight Pattern（享元模式）                        │
│      ─────────────────────────────────────                  │
│      用 TKey 作为技能ID而不是完整对象                        │
│      减少内存占用，便于在字典中快速查找                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 各模式解决的问题

| 模式 | 解决的问题 |
|------|------------|
| Observer | 索引自动响应技能变化，无需手动同步 |
| Strategy | 支持多种索引策略（单键/多键/自定义） |
| Factory Method | 封装索引创建细节，简化使用 |
| Flyweight | 用 ID 代替完整对象，减少内存 |

## 六、使用指南

### 6.1 基础使用示例

```csharp
// 场景：管理游戏中的技能

public class GameSkillManager
{
    // 技能库：技能ID → 技能数据
    private SkillLibrary<SkillId, SkillData> _skillLibrary;

    // 索引：按学院查询
    private DerivedKeyedSkillIndex<School, SkillId, SkillData> _bySchool;

    // 索引：按标签查询
    private DerivedMultiKeySkillIndex<int, SkillId, SkillData> _byTag;

    // 索引：按冷却时间查询
    private DerivedKeyedSkillIndex<CooldownTier, SkillId, SkillData> _byCooldown;

    public void Initialize()
    {
        _skillLibrary = new SkillLibrary<SkillId, SkillData>();

        // 创建索引
        _bySchool = _skillLibrary.CreateDerivedKeyedIndex<School>(
            d => d.School);

        _byTag = _skillLibrary.CreateDerivedMultiKeyIndex<int>(
            d => d.Tags ?? Array.Empty<int>());

        _byCooldown = _skillLibrary.CreateDerivedKeyedIndex<CooldownTier>(
            d => GetCooldownTier(d.CooldownMs));
    }

    // 注册技能
    public void RegisterSkill(SkillId id, SkillData data)
    {
        _skillLibrary.Add(id, data);
    }

    // 查询：所有火系技能
    public List<SkillData> GetFireSkills()
    {
        return _bySchool.Get(School.Fire)
            .Select(id => _skillLibrary.GetData(id))
            .ToList();
    }

    // 查询：所有带"控制"标签的技能
    public List<SkillData> GetControlSkills()
    {
        return _byTag.Get(SkillTags.Control)
            .Select(id => _skillLibrary.GetData(id))
            .ToList();
    }

    // 查询：火系+控制类技能
    public List<SkillData> GetFireControlSkills()
    {
        var fireSkills = _bySchool.Get(School.Fire);
        var controlSkills = _byTag.Get(SkillTags.Control);
        return fireSkills.Intersect(controlSkills)
            .Select(id => _skillLibrary.GetData(id))
            .ToList();
    }

    // 技能升级（索引自动同步）
    public void UpgradeSkill(SkillId id, SkillData newData)
    {
        _skillLibrary.Update(id, newData, new SkillUpdate(
            type: 1,  // 升级事件类型
            payload: new UpgradePayload(newData.Level)
        ));
    }
}
```
