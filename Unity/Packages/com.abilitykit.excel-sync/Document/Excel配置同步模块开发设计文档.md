## 一、设计理念：为什么需要配置同步？

### 1.1 配置管理的挑战

```
❌ 传统做法的痛点：

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  【方案一：纯 Excel 配置】                                             │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  优点：                                                          │  │
│  │  -策划可以独立编辑，不需要程序员参与                            │  │
│  │  -版本控制友好（文本格式）                                     │  │
│  │  -批量编辑方便                                                 │  │
│  │                                                                  │  │
│  │  缺点：                                                          │  │
│  │  -无法在 Unity 中预览效果                                      │  │
│  │  -类型安全差，需要额外的序列化/反序列化                         │  │
│  │  -难以实现复杂的数据验证和编辑器扩展                            │  │
│  │  -复杂嵌套结构（如技能配置）编辑困难                            │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  【方案二：纯 ScriptableObject 配置】                                  │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  优点：                                                          │  │
│  │  -Unity 原生支持，可在编辑器中实时预览                        │  │
│  │  -类型安全，编译时检查                                         │  │
│  │  -可使用 Odin 等工具实现复杂的编辑器界面                      │  │
│  │  -可引用其他资源（如 Prefab、材质等）                         │  │
│  │                                                                  │  │
│  │  缺点：                                                          │  │
│  │  -策划需要 Unity 编辑器使用知识                                │  │
│  │  -版本控制不友好（二进制格式或大量小文件）                     │  │
│  │  -批量编辑困难                                                 │  │
│  │  -多人协作时容易产生冲突                                        │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 二、Excel vs ScriptableObject 对比分析

### 2.1 特性对比表

| 特性 | Excel | ScriptableObject | 说明 |
|------|-------|-----------------|------|
| **策划友好度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | Excel 是策划最熟悉的工具 |
| **类型安全** | ⭐⭐ | ⭐⭐⭐⭐⭐ | ScriptableObject 有编译时检查 |
| **版本控制** | ⭐⭐⭐⭐⭐ | ⭐⭐ | Excel 可文本对比，SO 困难 |
| **批量编辑** | ⭐⭐⭐⭐⭐ | ⭐⭐ | Excel 复制粘贴方便 |
| **实时预览** | ⭐ | ⭐⭐⭐⭐⭐ | ScriptableObject 可在编辑器预览 |
| **复杂结构** | ⭐⭐ | ⭐⭐⭐⭐⭐ | SO 支持嵌套对象、引用 |
| **数据验证** | ⭐⭐ | ⭐⭐⭐⭐ | SO 可实现复杂验证逻辑 |
| **热更新支持** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | Excel 更易热更新 |
| **多人协作** | ⭐⭐⭐ | ⭐⭐ | 各有优劣 |
| **工具链复杂度** | ⭐⭐⭐ | ⭐⭐⭐⭐ | 需要导入导出工具 |

### 4.2 导入代码流程

```csharp
public static void ImportToSingleAssetDataList(
    ScriptableObject targetAsset,    // 目标 SO
    string excelFilePath,            // Excel 路径
    ExcelTableOptions options,        // 表配置
    ITableReaderWriterFactory factory,
    ExcelCodecRegistry registry)
{
    // 1. 验证目标 Asset 是否有 DataList 字段
    var dataListMember = FindDataListMember(targetAsset.GetType());
    var elementType = GetListElementType(dataListMember);

    // 2. 创建 Excel 读取器
    using var reader = factory.CreateReader(excelFilePath, options);

    // 3. 读取并验证表头
    var headers = reader.GetHeaders();
    ValidateHeadersStrict(elementType, headers, options, "Import");

    // 4. 构建列绑定
    var bindings = ExcelReflectionMapper.BuildBindings(elementType, headers);

    // 5. 遍历数据行
    var list = (IList)Activator.CreateInstance(typeof(List<>).MakeGenericType(elementType));
    var baselineRows = new Dictionary<string, List<string>>();

    foreach (var row in reader.ReadRows(options.DataStartRowIndex))
    {
        // 创建实例
        var item = Activator.CreateInstance(elementType);

        // 反射设置字段值
        foreach (var b in bindings)
        {
            var value = ExcelReflectionMapper.ConvertCellValue(
                row[b.ColumnIndex],
                b.ValueType,
                b.ColumnName,
                registry);

            ExcelReflectionMapper.SetValue(item, b.Member, value);
        }

        list.Add(item);

        // 记录 Baseline
        if (primaryKeyColumnIndex >= 0)
        {
            baselineRows[pk] = row.ToList();
        }
    }

    // 6. 设置到 DataList
    SetMemberValue(targetAsset, dataListMember, list);

    // 7. 保存 Baseline
    var baselineAsset = GetOrCreateBaselineAsset(targetAsset);
    baselineAsset.Set(excelFilePath, options, headers, baselineRows);
}
```

│  │   → Local 写入 Excel（策划只改了 SO）                          │  │
│  │                                                                  │  │
│  │   Base != Local  &&  Base == Remote                           │  │
│  │   → Local 写入 Excel（策划只改了 Excel）                       │  │
│  │                                                                  │  │
│  │   Base != Local  &&  Base != Remote                            │  │
│  │   && Local == Remote                                           │  │
│  │   → 无冲突（双方改了相同值）                                    │  │
│  │                                                                  │  │
│  │   Base != Local  &&  Base != Remote                            │  │
│  │   && Local != Remote                                           │  │
│  │   → ⚠️ 冲突！需要人工解决                                       │  │
│  │                                                                  │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  【冲突报告格式】                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                                                                  │  │
│  │  {                                                             │  │
│  │    "Items": [                                                 │  │
│  │      {                                                         │  │
│  │        "Key": "Skill_1001",                                   │  │
│  │        "Column": "Damage",                                     │  │
│  │        "Address": "Sheet1!D5",                                │  │
│  │        "Base": "100",                                         │  │
│  │        "Local": "150",        // SO 中的值                    │  │
│  │        "Remote": "120"         // Excel 中的值                │  │
│  │      }                                                         │  │
│  │    ]                                                           │  │
│  │  }                                                             │  │
│  │                                                                  │  │

namespace AbilityKit.Skills.Config

        public int Id;

        public string Name;

## 九、设计总结

### 9.1 核心优势

| 特性 | 说明 |
|------|------|
| **双向同步** | 支持 Excel → SO 和 SO → Excel 双向同步 |
| **冲突检测** | Baseline 机制实现三向比对，检测冲突 |
| **类型安全** | 基于反射的自动映射，保持类型安全 |
| **编辑器集成** | 无缝集成 Unity 编辑器，可实时预览 |
| **Partial 扩展** | 支持复杂类型的字符串编辑 |
| **版本控制友好** | Excel 可文本对比，减少协作冲突 |

### 9.2 扩展点

1. **新的类型支持**：扩展 `ExcelCodecRegistry`
2. **新的读取后端**：实现 `ITableReaderWriterFactory`
3. **数据验证**：扩展 `ValidateHeadersStrict` 方法
4. **冲突解决UI**：开发冲突解决编辑器窗口

### 9.3 注意事项

- **主键必需**：每个配置表必须有主键列
- **Baseline 重要**：不要删除 `.excelBaseline.asset` 文件
- **类型匹配**：Editor Partial 类和 Runtime 类字段名必须一致
- **热更新类型**：热更新层自定义类型需要特殊处理
